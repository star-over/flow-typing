# FlowTyping — доменный контекст

Адаптивный тренажёр слепой печати: «визуализация движения» пальцев + подача упражнений (drill) под уровень пользователя. Глоссарий единого языка проекта.

## Language

### Контент

**Drill**:
Атомарное задание для набора — одно слово или одно предложение.
_Avoid_: текст, урок, lesson

**Drill Corpus**:
Полное множество drill'ов, доступное системе для выдачи; хранит только layout-нейтральную мету в символьном пространстве.
_Avoid_: база текстов, корпус текстов

**Session (сессия)**:
Ограниченный таймером отрезок тренировки, внутри которого непрерывно подаются drill'ы; длительность — настройка пользователя; завершается по истечении таймера на границе текущего drill'а.
_Avoid_: lesson (внутреннее имя завершения одного drill'а в training.machine), урок

### Раскладки и пространства

**Layout Context**:
Среда печати, в которой осмысленна моторика: символьная раскладка × схема постановки пальцев (физическая — инвариант `ansi`). Операционно идентифицируется одним `symbolLayoutId`: пальцевая раскладка в системе одна (`asdf`) и существует только как фоновое допущение, заложенное в справочники классификации и KeyLadder (у которой есть `version`); отдельного поля в данных нет.
_Avoid_: «раскладка» без уточнения слоя, составные ключи вида `йцукен_asdf`, поля-штампы пальцевой раскладки

**Symbol Space (символьное пространство)**:
Характеристики в терминах символов текста; layout-нейтральны, принадлежат drill'у.

**Motion Space (моторное пространство)**:
Характеристики в терминах клавиш и пальцев (same-finger переходы, чередование рук); существуют только внутри Layout Context, однозначно выводятся из символьных.
_Avoid_: смешение «биграмма текста» / «биграмма движения» без указания пространства

### Модель пользователя

**Skill Profile**:
Всё знание системы о пользователе в одном Layout Context: ячейки статистики (per-символ, per-биграмма), Repertoire, позиции Difficulty Knobs, Fresh Window.
_Avoid_: user level, рейтинг

**Skill Level**:
Витринная оценка уровня (производная от скорости) для UI и мотивации; в подборе drill'ов не участвует — селектор читает Difficulty Knobs и Weakness Map.
_Avoid_: использование как механизма отбора; score, rating

**Difficulty Knobs (ручки трудности)**:
Хранимые в Skill Profile позиции ручек общей трудности (v1: целевая длина слова); термостат двигает каждую независимо.
_Avoid_: единый «уровень сложности», пресеты уровней

**Thermostat (термостат)**:
Правило адаптации общей трудности: лексикографический детектор зоны (приоритет точности → стабильность ритма и латентности) с гистерезисом и асимметрией (вниз быстро, вверх медленно); срабатывает на fetch-границах.
_Avoid_: Performance Score (похоронен)

**Fresh Window (свежее окно)**:
Кольцо сводок последних N drill'ов (accuracy, латентность, разброс) в Skill Profile — портрет «сейчас» для термостата, в отличие от ячеек-EWMA (портрет «по паттернам»).

**Weakness Map**:
Набор подтверждённых слабостей пользователя с оценкой выраженности; слабость — паттерн со статистикой значимо хуже собственной медианы пользователя (относительный критерий) при достаточном числе предъявлений; вход по порогу, выход с гистерезисом; в фокус заявки идут top-M по выраженности.
_Avoid_: Difficulty Vector (отменённый термин docs/05), абсолютные пороги («accuracy < 90%»)

**Repertoire (репертуар)**:
Множество введённого пользователю (символы, доступные через открытые клавиши); кодируется одним числом `openedSteps` — сколько шагов KeyLadder открыто (символ введён, когда весь его аккорд клавиш в пределах открытых шагов). Отвечает за факт ввода («ставили ли движение»); качество исполнения — зона Weakness Map.
_Avoid_: «освоенные символы» (степень освоения = качество, это Weakness Map), unlocked letters

**Readiness (готовность символа)**:
Параметризованный критерий по ячейке профиля: достаточно предъявлений + first-try accuracy выше порога + латентность не хуже k× медианной по репертуару. Условие расширения репертуара.
_Avoid_: «выучен», mastered (абсолютных порогов освоения нет — критерий относительный)

### Подбор

**DrillRequest (заявка на drill)**:
Декларативные требования к следующему drill — диапазоны характеристик, обязательные и запрещённые элементы; строится из Skill Profile, исполняется подбором по Drill Corpus.
_Avoid_: «сложность drill'а» как хранимый скаляр, Difficulty Vector

**Exposure (предъявление)**:
Одна встреча пользователя с целевым символом (один `StreamSymbol`), независимо от числа нажатий внутри; чистое предъявление — взят с первого нажатия. Знаменатель всех оценок профиля.
_Avoid_: «попытка» (в коде attempt = одно нажатие, не предъявление)

**Проскок (skip-ahead)**:
Артефакт ввода: пользователь, глядя на поток, а не на клавиши, при ошибке продолжает набирать следующие символы, не замечая её — застрявший символ копит ошибочные нажатия, которые на деле суть цели следующих позиций. Настоящая ошибка — только первое ошибочное нажатие; на реальных данных проскок ≈ 45% всех ошибочных нажатий. `summarize` обязан его вычитать; это же основа будущей подсказки «промах только в одной букве».
_Avoid_: считать каждое ошибочное нажатие путаницей

**Интерференция раскладок**:
Класс путаницы: нажата клавиша, где целевой символ (или его аналог в другой раскладке) живёт по мышечной памяти. Напр. `.`→Period (точка стандартной позиции вместо Slash на йцукен), английский `e`→позиция йцукен-`е`. Детектируется через справочники классификации; на реальных данных — самый воспроизводимый сигнал слабости.
_Avoid_: смешение с промахом по соседней клавише и с проскоком

**Batch (порция)**:
Результат исполнения одной DrillRequest: набор drill'ов с бюджетом в символах (≈ окно времени печати × cpm) и целью по доле фокусных (`focusShare` — цель, не жёсткая квота); количество drill'ов — производная бюджета, не параметр.
_Avoid_: batchSize как входной параметр (штуки ничего не обещают при разбросе длин), `focusShare` как обязательство (не набрал фокусных → не провал)

**DrillSummary (сводка drill'а)**:
Компактная дельта завершённого drill'а (по затронутым ячейкам: предъявления, чистые, латентности), вычисляемая чистой функцией `stream → DrillSummary` на клиенте: вычитает проскок, исключает первый символ drill'а из латентности, обрезает паузы; payload мутации `recordDrill`.
_Avoid_: пересылка сырых attempts на сервер

**Aggregator (агрегатор)**:
Двухчастный: `summarize` (клиент: `stream → DrillSummary`) + `apply` (Convex-мутация: транзакционно вносит сводку в ячейки профиля с затуханием); срабатывает на каждом завершении drill'а.

**Analyzer (анализатор)**:
Читает Skill Profile через справочники классификации, строит Weakness Map; запускается лениво — при формировании DrillRequest.

**Classification Table (справочник классификации)**:
Статичная таблица per Layout Context («символ → палец», «биграмма → тип движения»); единственное место, где живёт знание о FingerLayout.

**KeyLadder (лестница клавиш)**:
Рукотворный справочник с версией (`version`) per Layout Context — порядок, в котором клавиши открываются ученику, нарезанный вручную на шаги по геометрии раскладки (ADR 0002). Единица — клавиша (`keyCapId`), не символ; клавиши одного шага открываются вместе. Задаёт смысл `openedSteps` у Repertoire и производного `stepLevel` у drill'а.
_Avoid_: «программа уроков»; Introduction Order (прежнее имя — порядок символов по частотности, отменён); частотный порядок

## Relationships

- **Skill Profile** принадлежит паре (пользователь × **Layout Context**); у одного пользователя профилей столько, сколько Layout Context'ов он тренирует
- Профиль — server-authoritative в Convex (ADR 0005); гость — анонимная identity, создаваемая лениво при первом старте тренировки; регистрация = явный перенос профиля
- Вся адаптивная логика (apply-агрегатор, Analyzer, Thermostat, построитель заявки, подбор) — серверная, внутри `recordDrill` / `getNextBatch`; клиенту остаются training machine, таймер, буфер и `summarize`
- **DrillRequest** — внутренний артефакт серверного `getNextBatch`, не сетевой payload; клиент передаёт только контекст (раскладка, остаток таймера, cpm)
- **Skill Profile** хранит два уровня детализации — per-символ и per-биграмма; грубые уровни (клавиша, палец, рука, моторный класс) — производные на чтении через **Classification Table**
- **Session** — ограниченная таймером последовательность **Drill**'ов; выносливость — свойство Session (таймер), не drill'а
- Размер **Batch** задаётся бюджетом времени печати (символы ÷ cpm), окно меньше Session — ради адаптации внутри сессии; хвост клиентского буфера не переживает Session
- **Aggregator** обновляет **Skill Profile** на каждом завершении drill'а; **Analyzer** строит **Weakness Map** лениво — на старте сессии и при дополнении буфера
- Движок адаптации строит **DrillRequest** из **Skill Profile**: Difficulty Knobs → ручки общей трудности, Weakness Map + неподтверждённые символы → фокус-ручки
- **Thermostat** читает **Fresh Window** и двигает **Difficulty Knobs**; каждый шаг объясним одним предложением и фиксируется в журнале
- **DrillRequest** исполняется по **Drill Corpus**: жёсткие требования — предварительная фильтрация, мягкие — ранжирование по близости
- **Repertoire** — единственное жёсткое требование **DrillRequest** (drill совместим, если его `stepLevel` в таблице отбора per Layout Context `< openedSteps`); фокус, длина, анти-повтор — мягкие, решаются ранжированием
- Деградация при тонком пуле кандидатов **возникает сама собой**, отдельной ветки «ослабления требований» нет: чем тоньше пул — тем хуже средняя близость выданного к заявке, но **Batch** всегда набирается. `focusShare` — цель (не хватает фокусных → добор обычными из того же ранжирования); недобор `batchChars` допустим (ранний поход за следующей порцией)
- Ноль кандидатов после жёсткого фильтра — не деградация, а **контентный сбой** (дыра, которую ловит контентный радар ADR 0002): дефолтный drill (домашний ряд) + лог-сигнал, сессия не блокируется
- Репертуар растёт шагами: следующий шаг KeyLadder открывается, когда символы текущего шага достигли **Readiness** (число не-готовых ≤ долгового лимита); недавно открытый символ получает фокус заявки как *неподтверждённый* — второй источник фокуса наряду со слабостями
- Шаг лестницы масштабируется качеством данных: по умолчанию +1 шаг KeyLadder; при подавляющей статистике Readiness наступает досрочно и ввод идёт блоками (ускоренная лестница опытного)
- Cold start универсален, без вступительного вопроса: новый профиль = стартовый шаг KeyLadder (шаг 0: указательные + пробел) + ручки на минимуме; траектории расходятся скоростью сходимости, а не ветвлением. Лестница опытного — одновременно его калибровка
- **Readiness** и слабость — зеркальные относительные предикаты над одними ячейками профиля: «достаточно хорош для расширения» / «достаточно плох для фокуса»
- Смена **KeyLadder** = миграция: `version`, пересчёт `openedSteps` пользователей, пакетный пересчёт таблицы отбора (`stepLevel`)
- **KeyLadder** строится по геометрии раскладки ручными шагами (ADR 0002), не по частотности; покрытие корпуса против шагов — контентная метрика («доступно drill'ов на шаге k»)
- **Motion Space**-знание раскладки компилируется в Symbol Space-множества per Layout Context (например, «same-finger биграммы на йцукен×asdf»), а не хранится per drill
- Мета drill'а не содержит триграмм (v1): нет потребителя в профиле; восстановимы из текста пакетно

## Example dialogue

> **Dev:** «Пользователь стал быстрее — двигаем ручки трудности вверх?»
> **Domain expert:** «Только если он в зоне скуки: точность над верхним порогом и ритм стабилен несколько окон подряд. А если при этом в Weakness Map появились новые биграммы — фокус-ручки следующего DrillRequest прорабатывают их на той же общей трудности.»

## Flagged ambiguities

- «Сложность drill'а» как заранее вычисленный атрибут — **отвергнута**: она layout-зависима и персональна. Роль распределена между ручками DrillRequest (целевая трудность) и мерой близости подбора (отклонение от заявки).
- «Performance Score» из docs/05 — **похоронен**: композитная свёртка с магическими весами заменена лексикографическим детектором зон (ADR 0004). Вместе с Difficulty Vector это второй умерший термин docs/05 — документ переписан под новую модель.
- «Difficulty Vector» из docs/05 смешивал сложность drill'а и фокус пользователя — **superseded** термином DrillRequest.
- «lesson» vs «session» — **разрешено**: Session = ограниченный таймером отрезок из многих drill'ов; `lessonComplete` в `training.machine` — внутреннее имя завершения одного drill'а (деталь реализации). Текущее «одна сессия = один drill» — переходное состояние (backlog «Session vs drill»).
