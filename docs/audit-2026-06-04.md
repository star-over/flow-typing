## Приоритет 2 — главная архитектурная дыра (несколько вечеров)

### 2.1. Тесты на XState-машины
Самое серьёзное упущение. Сейчас покрыта только `keyboard.machine.test.ts` (10 тестов). А `app.machine.ts` и `training.machine.ts` — это **вся бизнес-логика проекта**, и они без юнит-тестов.

**В первую очередь:**
- `training.machine`: правильно ли копятся `attempts` при ошибках, корректность `areKeyCapIdArraysEqual` при наборе модификаторов, переход в `COMPLETE` после последнего символа.
- `app.machine`: guard'ы на Escape/Enter, что URL ↔ store sync не теряется при `RESTART_TRAINING`, что `currentKeyboardLayout` правильно прокидывается в `START_TRAINING`.

### 2.2. Тесты на `difficulty-calculator.ts`
Адаптивная система (см. `docs/05-adaptive-learning-system.md`) — математическое ядро продукта. Сейчас оно **без единого теста**. Стоимости пальцев (`LB=99`), модификаторы, граничные случаи. Если таблица сломается — «адаптивность» тихо превратится в обычный clickity-clack.

### 2.3. Унификация хардкода раскладки в `app.machine`
В `app.machine.ts` `keyboardLayoutANSI` импортируется напрямую и засовывается в `invoke` для `keyboardMachine` — это **физическая** геометрия (инвариант). А `currentKeyboardLayout` (qwerty/йцукен) идёт из preferences — это **символьная** раскладка. Концепции разные, но в коде они не названы, и CLAUDE.md приходится отдельным абзацем объяснять «это разные слои».

**Что сделать:** переименовать импорт в `PHYSICAL_KEYBOARD_LAYOUT` (или константа наверху файла) + один однострочный комментарий. Эта путаница регулярно цепляет глаз.

## Приоритет 3 — UX и мобильный опыт

### 3.1. Mobile сломан
`width: 100vw` в `HandsExt.svelte` и `FlowLine.svelte` + фиксированные `36px` keycap-unit, `1.875rem` для текста. На телефоне руки уезжают за вьюпорт. Если в планах не «только десктоп» — первое, что заметит любой посетитель.

**Что сделать:** заменить `100vw` на `100%` с `max-width` контейнером, добавить `@media (max-width: 768px)` в `app.css` с уменьшением `--keycap-unit`, использовать `clamp()` для `font-size` в FlowLine.

### 3.2. Edge cases в потоке
`FlowLine.svelte:44–58` обращается к `stream[cursorPosition]` без guard на конец потока. С `noUncheckedIndexedAccess: true` это должно подсветиться по типам — проверить, не маскируется ли через `!`. Добавить явный fallback для пустого / завершённого потока.

### 3.3. A11y минимум
- `Select.svelte` без `aria-label`.
- Фокус не переносится при смене экрана (menu → settings → training).
- Нет `aria-live` для смены состояния.

Не критично для типинг-тренажёра, но ~30 минут работы.
