# Диакритика через dead-key: цель набора — последовательность аккордов

> Статус: предложен · 2026-07-04 · Реализация: — · Опирается на границу слоёв ADR 0016.

При добавлении испанского/португальского (и части французского) частые буквы
набираются «мёртвой клавишей» (dead key): нажать `´`, затем `e` → `é` — **две
последовательные** клавиши. Сейчас `StreamSymbol.targetKeyCaps: KeyCapId[]` и
`SymbolLayout.keyCaps` моделируют **одновременный аккорд** (база + Shift/AltGr,
зажатые вместе), а не темпоральную цепочку. Это единственная реальная нехватка
доменной модели при расширении на европейскую латиницу (немецкий и французский для
своих основных букв обходятся отдельными клавишами на ISO и dead-key не требуют —
фаза 1 не заблокирована; фича нужна под испанский/португальский).

Предлагаемое решение: расширить представление цели символа с «один аккорд» до
**последовательности аккордов** — цель может быть списком шагов, каждый шаг — аккорд;
одиночный символ становится частным случаем (один шаг). Статус «предложен»:
финализировать при скоупинге испанской раскладки, не задним числом.

## Considered options

- **Не поддерживать dead-key, красить акцентированные буквы отдельными клавишами** —
  отвергнут: не соответствует раскладкам, которыми люди реально печатают; ломает
  достоверность обучения моторике.
- **Моделировать dead-key как один расширенный аккорд** — отвергнут: физически это
  два раздельных нажатия во времени; аккорд (одновременность) семантически неверен, и
  «путь пальца» должен показать два движения, а не одно.
- **Выбран (предложен): цель = последовательность аккордов** (шагов); одиночный
  символ — один шаг.

## Consequences

- Меняется форма `targetKeyCaps` и производных: hands-scene pipeline, сравнение в
  `trainingMachine`, `FlowLine`. Сквозное изменение — потому ADR, а не деталь кода.
- Классификатор `trainingMachine` продвигается по шагам **внутри** символа, не только
  по символам.
- Визуализация движения получает второй сегмент пути (dead-key → буква) — согласуется
  с философией «движение важнее подсветки».
- Не блокирует немецкий/французский (фаза 1); требуется перед испанским/португальским.
- Решение не ратифицировано (предложен) — при принятии обновить статус и индекс
  `README.md`.
