# Форма записи символьной раскладки объявлена трижды — ради развязки shared↔src

Форма записи раскладки (`{ symbol, keyCaps }`) объявлена в трёх местах — `shared/symbol-layout.ts` (`interface SymbolEntry`, `keyCaps: string[]`), `src/interfaces/types.ts` (`type SymbolLayout`, `keyCaps: KeyCapId[]`) и `src/lib/layouts.ts` (zod `SymbolEntrySchema` через `KeyCapIdSchema`). Расхождение формы компилятор не поймает. Решение: **принять дрейф**, не сводить к единому владельцу — три объявления служат трём разным мастерам, а единый владелец невозможен без слома активного инварианта.

## Контекст: три мастера, не одна дупликация

- **`shared/symbol-layout.ts` — серверно-tooling-форма.** Слабый `keyCaps: string[]`, потому что её импортируют `convex/layoutData.ts`, `auto-flow/symbol-layout.ts`, `shared/repertoire`, `shared/key-ladder` — у них нет `KeyCapId` (литеральный union живёт только в `src`).
- **`src/interfaces/types.ts` — доменный словарь.** Сильный `keyCaps: KeyCapId[]` с рукописным JSDoc, который `CLAUDE.md` защищает как «часть документации единого языка, не удалять».
- **`src/lib/layouts.ts` — рантайм-гейт.** zod парсит недоверенный JSON раскладки на входе.

## Решения

1. **Не унифицируем.** Три объявления сохраняются как есть.
2. **Развязка `shared`↔`src` — активный инвариант, не совпадение.** `src` не импортирует `shared` **ни разу** (проверено grep'ом; комментарии в `repertoire-store.svelte.ts` / `progression-store.svelte.ts` прямо это запрещают). `shared` остаётся без прод-типов, чтобы его могли безопасно тянуть сервер (Convex) и tooling (auto-flow).

## Considered options

Полный единый владелец требует заплатить одну из трёх цен — каждая хуже самого дрейфа:

- **`shared` импортирует `KeyCapId` из `src`** — ломает развязку (shared начинает знать про src).
- **`src` импортирует `shared`** — ломает развязку и завязывает фронт на слабый `string[]`.
- **Ослабить `src` до `keyCaps: string[]`** — теряется типобезопасность `KeyCapId` во всём фронте.

Частичная унификация внутри `src` (`type = z.infer` из zod-схемы) границу бы не пересекла, но инвертировала бы зависимость (доменный словарь начал бы зависеть от парсера) и заменила бы JSDoc-тип на выведенный — против конвенции «`types.ts` = единый язык». Тоже отвергнута.

## Consequences

- Расхождение формы через границу компилятор не ловит — смягчается тем, что форма структурно тривиальна (`symbol: string` + массив ключей) и каждый вход в `src` перепроверяется рантайм-гейтом (`SymbolEntrySchema`).
- Зафиксировано в аудите тест-системы (`docs/plans/2026-07-03-test-system-audit.md`, секция C).
