# Skill Profile — server-authoritative в Convex; клиент тонкий

docs/05 рекомендовал «гибридный local-first» (профиль и логика на клиенте, фоновая синхронизация). Решение противоположное: профиль живёт только в Convex, вся адаптивная логика (применение сводок, анализатор, термостат, построитель заявки, подбор) — серверная; клиент держит training machine, таймер, буфер порций и чистую функцию `stream → DrillSummary`. Ключевой довод: корпус всё равно в Convex, сеть уже обязательна для тренировки — а server-authoritative профиль не откладывает проблему синхронизации устройств, он её **устраняет** (один писатель, одна истина, транзакции из коробки).

## Considered options

- **Client-first + отложенная фаза синхронизации** — прятал самое трудное (merge двух мутирующих профилей) в «потом»; оффлайн-довод фиктивен (без сети нет и drill'ов); тестируемость не аргумент — паттерн handler + convex-test уже в проекте.
- **Гибрид «клиент считает, Convex хранит»** — два писателя при двух устройствах, гонки снимков состояния.

## Consequences

- Гость = анонимная identity (`@convex-dev/auth` Anonymous), создаётся лениво при первом старте тренировки; регистрация = явная мутация переноса профиля (инвариант «провайдер = аккаунт» не трогается — это не link-by-email).
- Агрегатор разрезан по шву тестирования: `summarize` (клиент, чистая, vitest-проект `src`) → `drillRecord` mutation (`apply`, vitest-проект `convex`). Payload — килобайтная сводка, не attempts. (Реализованы под именами семейства `drill*`: `drillRecord`, `drillNext` — единый общий префикс.)
- DrillRequest — внутренний артефакт серверного `drillNext` (профиль → заявка → подбор в одной транзакции), клиент шлёт только контекст (раскладка, `openedSteps`, бюджет в символах; cpm/таймер остаются на клиенте — уточнено ADR 0006). Читаемость заявки сохраняется записью в журнал.
- Маленький localStorage-outbox для неотправленных сводок — очередь доставки (at-least-once), не система синхронизации.
- Развёртывание движка адаптации не ждёт обновления клиентов; кросс-девайс работает с первого дня.
- Оффлайн-тренировка официально вне scope v1.
