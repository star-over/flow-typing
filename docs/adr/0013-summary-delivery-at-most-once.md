# Доставка сводок — at-most-once; обещание outbox снято; повторная доставка — только в паре с идемпотентностью

> Статус: принят · 2026-07-03 · Поправляет: ADR 0005 (снимает consequence об outbox) · Реализация: семантика уже в коде (fire-and-forget); правка комментария в `session-impl.ts` — вместе со сносом гостевых веток (ADR 0012)

ADR 0005 обещал «маленький localStorage-outbox для неотправленных сводок
(at-least-once)». Обещание не реализовано, и по аудиту
(`docs/plans/2026-07-03-adr-canon-audit.md` §4.2) оно конфликтует с семантикой
писателя: `foldSummaryIntoCells` строго аддитивен, защиты от дублей нет —
повторная доставка той же сводки удвоила бы предъявления и исказила вход
Readiness/термостата. Наивный outbox по букве ADR 0005 портил бы профили.

Решение: **доставка сводок — официально best-effort (at-most-once)**, для обеих
записей единообразно — чекпоинт `drillRecord` и журнал `sessions.record`. Потеря
дельты при сбое сети допустима: профиль — статистическая проекция, не бухгалтерия;
недобранные предъявления доберутся в следующих сессиях (система восстанавливается
сама), журнал теряет одну строку `/stats`. Краевой случай узкий:
сеть должна упасть посреди сессии — без сети сессия не начинается вовсе (офлайн вне
scope v1, ADR 0005/0011).

**Предохранитель (несущая часть решения):** если когда-либо появляется повторная
доставка — outbox, retry, офлайн-очередь — она обязана прийти **в паре** с
идемпотентностью apply (ключ чекпоинта в payload + серверный учёт обработанных
ключей). Поодиночке эти механизмы запрещены: повторная доставка без идемпотентности
портит профили, идемпотентность без потребителя — мёртвая машинерия.

## Considered options

- **Outbox + идемпотентность по букве ADR 0005** — честный at-least-once без потерь,
  но реальная машинерия (очередь в localStorage, политика повторов, серверное
  хранение и уборка ключей, гонки двух вкладок над одной очередью) ради краевого
  случая, который v1 объявил вне scope. Отвергнут сейчас; зафиксирован как
  единственно допустимая форма будущего возврата (пара, не поодиночке).
- **Идемпотентность впрок, без outbox** — «дешёвая страховка», но источника дублей
  сегодня нет (одна fire-and-forget отправка из action машины), а хранение ключей
  без потребителя — спекулятивная машинерия; проект такое уже отвергал (ADR 0009:
  soft-слой «не материализуем, пока нет реального сигнала»). Отвергнута.

## Consequences

- Consequence ADR 0005 об outbox снят (пометка в статус-строке 0005); код не
  меняется — fire-and-forget с молчаливым гашением и есть принятая семантика.
- Комментарий «гость/офлайн» у silent-catch в `session-impl.ts` при сносе гостевых
  веток (ADR 0012) сужается до «офлайн: сводка теряется by design (ADR 0013)».
- CONTEXT.md (Aggregator) фиксирует: доставка сводки — best-effort (at-most-once).
- Драйвер пересмотра: появление офлайн-scope или реальные жалобы на пропавшие
  строки `/stats` → возврат строго парой «outbox + идемпотентность», отдельным ADR.
