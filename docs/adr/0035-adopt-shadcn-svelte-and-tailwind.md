# Хром — на shadcn-svelte + Tailwind v4; домен остаётся bespoke

> Статус: предложен · 2026-07-23 · Поправляет: ADR 0033 · В силе: ADR 0029, 0034 · Реализация: в работе

## Context

ADR 0033 отклонил готовую библиотеку компонентов одной строкой: «shadcn =
React+Tailwind, назван в запрете `CLAUDE.md`». Обоснование устарело: `shadcn-svelte`
(Bits UI + Tailwind v4) существует, работает на Svelte 5 и не тянет React. Запрет
«CSS без фреймворков» тем самым лишился своей причины на слое хрома.

Что заставило пересматривать, а не оставить как есть: из 73 ролей темы 34% — хром,
и он даёт непропорционально много ручной работы (кнопки, поля, именование, состояния,
тёмная тема). Траектория проекта — выделение библиотеки компонентов; для библиотеки
готовая структура (варианты, состояния, доступность, тёмная тема) не мелочь, а главная
ценность. Конкретный дефект, который bespoke не закрывал: нативный `<select>` рисует
светлое всплывающее меню в тёмной теме и не темизируется в принципе — таких `<select>` в проекте два.

Прежний контрдовод «Tailwind вредит приватности offline-first SPA» снят: Tailwind v4 —
генерация обычного CSS на этапе сборки, без рантайма и без сети.

Интеграция доказана эмпирически (ветка `spike/shadcn-select`): сборка и проверки
зелёные, темы целы в скомпилированном CSS, `Select` — настоящий составной список Bits UI.

## Decision

1. **Хром переходит на `shadcn-svelte` (Bits UI + Tailwind v4).** Примитивы `ui/`
   (`Button`, `Select`, `Card`, `Field`, `Input`, `Avatar`, `MessageScreen`→`empty`,
   `KeyHint`→`kbd`) и ручные каркасы (`UserMenu`, danger-подтверждение,
   `SessionDurationSelector`) строятся на компонентах реестра. `Wordmark` — бренд,
   остаётся bespoke.
2. **Решения ADR 0033 остаются в силе**, меняется только их реализация. Шесть
   семантических вариантов, полиморфизм `href→<a>`, foreground плотных на
   `--color-on-dense`, ось размера — переносятся на `tailwind-variants`.
3. **Tailwind разрешён во всём `src/**`**, включая доменные компоненты. Границы
   «хром/домен» в правилах нет.
4. **Домен визуализации движения остаётся bespoke** — не покрывается библиотекой ни
   при каком раскладе. Вычисляемые цвета (`oklch(from …)`), маски, keyframes и
   SVG-геометрия остаются CSS. Гейты ADR 0029 (роли) и 0034 (честный вывод, ΔE/WCAG)
   продолжают действовать без изменений.
5. **Две-словарная модель токенов.** Домен-роли — единственная истина под
   контракт-тестом (`roles.ts`, `contract.test.ts`/`resolved.test.ts`); словарь имён-ключей
   shadcn — их проекция для утилит Tailwind. Namespace `--color-*` принадлежит Tailwind,
   поэтому роли переименовываются в **`--role-*`**, а `@theme inline` проецирует их под
   именами, которые нужны утилитам. Разведение по имени делает два словаря различимыми
   и снимает коллизию имён-ключей `background`/`border` с одноимёнными ролями.
6. **Инвариант порядка `@import`.** Файлы тем импортируются **до** `tailwindcss`.
   Иначе развёрнутое им non-import содержимое `@layer` делает недействительными последующие
   `@import` (спецификация CSS), и все темы молча выпадают из прод-сборки при зелёном
   `make build`. Держится гейтом по скомпилированному CSS, не тестом источника.

## Considered options

- **Оставить bespoke (статус-кво ADR 0033)** — отвергнут: не закрывает светлое всплывающее меню
  `<select>` в тёмной теме (нативный виджет не темизируется), оставляет ~34% ролей на
  ручном сопровождении и не даёт готовой структуры под будущую библиотеку.
- **shadcn только внутри `ui/`** — отвергнут в пользу Tailwind везде (владелец): при
  запертых утилитах домен всё равно ссылается на роли, но именование ролей как палитры
  Tailwind даёт `bg-finger-1` без потери контракта, а граница-по-каталогу требует
  ручного надзора вместо машинной проверки.
- **Самоссылка ролей вместо переименования** (`--color-x: var(--color-x)`) —
  отвергнута как конечное состояние: работает (проверено на сборке), но
  порождает циклические декларации и делает корректность зависимой от приоритета слоёв;
  используется лишь как временные леса до шага переименования.
- **shadcn + Tailwind (выбран)** — снимает исходную боль хрома из коробки, оставляет
  домен нетронутым под его гейтами, обычный CSS на выходе (приватность цела).

## Consequences

- Запрет «CSS без фреймворков · Никаких Tailwind, shadcn» в `CLAUDE.md` снят; вводятся
  новые deps (`tailwindcss`, `@tailwindcss/vite`, `bits-ui`, `shadcn-svelte` и спутники).
- Появляется второй словарь токенов. Он различим по namespace (`--role-*` наше,
  `--color-*` — проекция Tailwind), но восемь пар остаются семантическими дублями
  (`--role-text-secondary` ↔ `--color-muted-foreground` и т. п.) — цена соседства с
  фиксированным словарём библиотеки.
- Все ссылки `var(--color-*)` в компонентах и темах переименовываются в `--role-*` —
  широкая механическая правка, целиком под контракт-гейтом.
- Появляется гейт по скомпилированному CSS (присутствие тем, привязка утилит
  типографики к `--font-size-*`) — класс регрессий, который источник-тесты не ловят.
- ADR 0033 сохраняет силу по существу (модель вариантов), теряет силу лишь его строка
  «готовая библиотека отвергнута».
