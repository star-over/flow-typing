# KeyLadder строится по геометрии раскладки (постановке пальцев), не по частотности языка

> Статус: принят · 2026-06-13 · Реализация: в коде (`shared/key-ladder/jcuken.ts`, version 1)
> История: 2026-06-15 тело переписано с обратным курсом (порядок по частотному словарю → ручные шаги по геометрии) — до политики неизменности (README)

FlowTyping — тренажёр с ядром «визуализация движения пальца от домашней позиции». Значит клавиши должны открываться ученику **по досягаемости и постановке пальцев от дома**, а не по частотности букв языка. Решение: **KeyLadder** — порядок открытия клавиш per Layout Context — **нарезается на шаги вручную** (`KeyLadderEntry.step`), опираясь на геометрию раскладки (finger-layout, физические ряды), а не на частотный словарь. Репертуар пользователя кодируется числом открытых шагов (`openedSteps`); единица лестницы — клавиша (`keyCapId`), не символ.

## Considered options

- **Чисто частотный по буквам (keybr-style)** — раскидывает новичка по рядам с первых же букв, до постановки рук, и игнорирует моторную трудность клавиши. Для тренажёра движения это неверная ось: частотность — про то, *что* печатать, а не про то, *как ставить пальцы*. Отвергнут.
- **Автоматическая геометрическая группировка алгоритмом** — «по пальцу» тянет дальние клавиши (палец охватывает все ряды, цифровой ряд открылся бы рано); «по рядам» слишком грубо; единый алгоритм не даёт контроля над учебной кривой. Отвергнут в пользу ручной нарезки.
- **Ручные шаги по пальцам/досягаемости (выбран)** — куратор расставляет `step` каждой клавише; геометрия — подсказка, не алгоритм. Полный контроль над программой, прозрачно, проверяется автоматически.

## Consequences

- KeyLadder — рукотворный статический справочник с версией per Layout Context (в коде), не runtime-компонент и не вывод из корпуса.
- Автоматическая проверка гарантирует инварианты: покрыты ровно все клавиши раскладки, шаги `0..max` без дыр, стартовый блок (`Space`) в шаге 0.
- «Контентный радар» остаётся: покрытие живого Drill Corpus меряется против шагов («доступно drill'ов на шаге k») — контентная метрика, на лестницу не влияет. Дыра = повод **добавить тексты**, а не ослаблять требования.
- Смена KeyLadder после запуска — миграция: `version`, пересчёт `openedSteps` пользователей, пакетный пересчёт таблицы отбора (`stepLevel`). См. ADR 0001.
- Частотный словарь языка больше не нужен (закрывает бывший открытый вопрос об источнике эталонного словаря).
