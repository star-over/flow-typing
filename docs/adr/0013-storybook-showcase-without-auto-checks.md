# Storybook — визуальная витрина без авто-проверок

12 компонентов (`SignInScreen`, `UserMenu`, `FlowLine`, `Finger`, `HandsScene`, `KeyboardScene`, `KeyCap`, `NavArrow`, `RhythmChannel`, `LessonStatsDisplay`, `Avatar`, `Wordmark`) живут только в Storybook и не получают **никакой** автоматической проверки: нет `@storybook/test`/`test-runner`, ноль play-функций, `make check-all` Storybook не гоняет, a11y-расширение работает лишь в интерактивном UI. Позиция «компоненты = только Storybook» уже декларирована в `CLAUDE.md`; этот ADR фиксирует её как **осознанную**, чтобы будущие аудиты не пере-предлагали play-функции / browser-mode.

## Решения

1. **jsdom/browser-mode в CI не вводим.** Проект `src` идёт в node-окружении, компоненты компилируются в server-mode (`$effect` вырезается), ни один тест не рендерит компонент. Это by design (`CLAUDE.md`, «Нет `jsdom`/`happy-dom`»).
2. **Витрина проверяется визуально.** Storybook — инструмент ручного визуального ревью и разработки в изоляции, не тестовая поверхность. Автоматической проверки рендера/поведения у компонентов нет намеренно.
3. **Логика — за чистыми швами, покрытыми в node.** Кристаллизуемые инварианты вынесены из компонентов в тестируемые `.ts` (`computeAuthState`, `computeTimerSeconds`, `registerBeatReducer`, `rhythm-channel.ts`, ViewModel-pipeline `hands-scene.ts` и т.д.). Тонкая руна поверх чистого шва не нуждается в компонентном тесте.

## Considered options

- **Добавить play-функции + `@storybook/test` (или portable-stories в vitest browser-mode)** — отвергнут: тянет jsdom/browser-runtime в CI (принципиально исключены), а «рендер без throw» проверяет мало сверх того, что уже даёт `svelte-check` + покрытые чистые швы. Цена инфраструктуры не оправдана для витрины.

## Consequences

- 12 story-only компонентов остаются без логического авто-теста — приемлемо: их логика вынесена за швы и покрыта, а визуал — предмет ручного ревью в Storybook.
- Если появится компонент с невыносимой из него логикой (не кристаллизуемой в чистый шов), эта позиция подлежит пересмотру — но по умолчанию логика выносится, а не тестируется через рендер.
- Зафиксировано в аудите тест-системы (`docs/plans/2026-07-03-test-system-audit.md`, кандидат 8).
