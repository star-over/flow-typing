# `drillNext` тотален: сервер всегда возвращает drills, дефолт на контентный сбой; клиентского корпуса нет

> Статус: принят · 2026-06-29 · Развивает: ADR 0002 (контентный радар), ADR 0009 (двухслойный отбор) · Реализация: в коде

Развивает ADR 0002 (контентный радар) и ADR 0009 (двухслойный отбор). Раньше политика контентного сбоя была расщеплена сервер↔клиент: сервер **сигналил** `{ contentGap: true, drills: [] }`, клиент **решал** деградировать на второй, клиентский корпус (`DRILL_CORPUS`, встроенный в браузерную сборку из `drills.jsonl?raw`, через `fetchLocalDrillStream`). Один политический вопрос — два дома; плюс второй корпус drill'ов, живущий на клиенте. Решение: сервер владеет политикой целиком и **тотален** — для раскладки с серверными данными всегда возвращает непустой `drills`, на контентный сбой подставляя дефолтный drill. Клиентский корпус и все ветки деградации удалены.

## Решения

1. **Сервер тотален.** `drillNext` всегда возвращает `drills` (непустой для раскладки с серверными данными). На контентный сбой — пустой пул (`count === 0`) ИЛИ сплошь битые ссылки — строит дефолтный drill из символов **открытых шагов** (steps `0..openedSteps-1`): это в точности символы, чьи drill'ы пропали, поэтому repertoire-безопасно и из источника истины KeyLadder (`symbolsAtStep`), без жёсткой записи. Раскладка без серверных данных (`getLayoutData === null`) → пусто: построить нечем, это config-баг, не норма.

2. **Контракт минимален.** `{ drills: [{ text }] }`. Убраны: булев `contentGap` (избыточен — дыру закрыл сервер, сигналу некому слушать; клиент и так читал только `drills.length`) и per-drill `id`/`length` (клиент читает только `.text`; внутренние `id`/`length` идут на отбор и бюджет, не на провод; синтетический дефолт `Id<'drills'>` не имеет).

3. **Клиентского корпуса нет.** Удалены `DRILL_CORPUS` (`drill-corpus.ts`), `fetchLocalDrillStream`, `drill-selection.ts` (`filterDrillsBySymbolLayout` / `selectRandomDrill`) и load-time проверка покрытия `integrity.ts`. Корпус drill'ов теперь server-authoritative (таблица `drills`, в духе ADR 0005) — единственный.

4. **Склейка-пробелом — один дом.** Правило «границы drill'ов невидимы, но разделены ровно одним пробелом» (CONTEXT.md: пробел внутри неотличим от стыка) сведено в `drillSeparatorStream`, потребляемый и склейкой внутри порции (`glueDrillsIntoStream`), и склейкой на стыке порций (`joinBatchToStream` → `appendFetched` в `session.machine.ts`). Раньше стык клеился отдельным сепаратором в машине.

5. **Офлайн не поддержан (этот этап).** Сервер недоступен → `fetchDrills` бросает → `sessionMachine` уходит в `done`. Локальной деградации нет.

## Considered options

- **Оставить клиентскую деградацию на contentGap (статус-кво)** — отвергнут: два дома у одной политики, второй корпус в сборке, избыточный сигнал (клиент игнорировал булев, ветвился по `drills.length`).
- **Сервер сигналит именованный contentGap, клиент решает (полиморфный union)** — отвергнут: единственным ответом клиента на contentGap была деградация на корпус; после удаления корпуса ответ полностью серверный, и у сигнала не остаётся потребителя на клиенте.
- **Дефолт строит клиент** — невозможен после удаления корпуса и противоречит принципу «на клиенте корпуса нет».
- **Сервер отдаёт фиксированную home-row строку на раскладку** — отвергнут: дублирует то, что уже кодирует KeyLadder, и требует ручной записи на каждую новую раскладку. Дефолт из символов открытых шагов выводится из источника истины.

## Consequences

- Инвариант покрытия съезжает с клиента. Load-time guard `integrity.ts` («у каждой раскладки реестра ≥1 совместимый drill») удалён; его повод (время выполнения «no drills for layout») поглощён серверным дефолтом. Покрытие — серверная/seed-забота (контентный радар ADR 0002).
- `contentGap` больше не наблюдаем как поле контракта — только как серверный `console.warn`. Доменный термин живёт в CONTEXT.md и этом ADR.
- Гостевой доступ не тронут (онлайн-гость получает cold-start drill'ы с сервера). Удаление неавторизованного доступа — отдельное решение.
- `drill.test.ts` проверяет по `.text` (без `id`/`length`); contentGap-тесты проверяют дефолтный drill (йцукен — единственная раскладка с серверными данными).
- Реализовано веткой `refactor/drill-source-server-total` (ADR-аудит, Этап 2).
