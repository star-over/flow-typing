# Зона потока определяется лексикографическими правилами, не композитным скором

docs/05 предлагал «Performance Score» — взвешенную свёртку точности, ритма и скорости в одно число. Отвергнуто: три магических веса нечем обосновать, единое число не объясняет решений системы, настройка слепая. Вместо этого зона (фрустрация / поток / скука) определяется лексикографическим детектором с приоритетом точности (философия проекта: точность > скорость): (1) accuracy свежего окна ниже порога → фрустрация; (2) accuracy выше верхнего порога И ритм стабилен И латентность не растёт → скука; (3) иначе — поток. Гистерезис: шаг ручек только после K окон в зоне; асимметрия: вниз быстро (фрустрация дороже), вверх медленно. Состояние трудности — не скалярный «уровень», а независимые позиции ручек (Difficulty Knobs) в профиле; витринный уровень для UI — производная от скорости, в подборе не участвует.

## Consequences

- Каждый шаг термостата объясним одним предложением — фиксируется в журнале и пригоден для debug-панели.
- Skill Profile дополняется «свежим окном» — кольцом сводок последних N drill'ов (портрет «сейчас», в отличие от ячеек-EWMA — портрета «по паттернам»).
- «Performance Score» и скалярный уровень-механизм похоронены вслед за «Difficulty Vector» (ADR 0003); docs/05 подлежит переписыванию под принятую модель.
