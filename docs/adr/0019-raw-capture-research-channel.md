# Сырьё печати — отдельный opt-out research-канал: gzip-блоб на сессию в File Storage + тонкая мета; не через профиль; time-boxed

> Статус: предложен · 2026-07-05 · Поправляет: ADR 0005 (при принятии) · Реализация: —

MVP по видению (`docs/01`) — проверка гипотезы и **калибровка чисел** адаптивной системы; этапы «Фокус»/«Термостат» (ADR 0004, 0018) сознательно ждут реальных данных печати. Но живые агрегации **не сохраняют полноту и заранее зафиксированы в проверяемые гипотезы**: `skillProfiles` складывает латентность через EWMA (`convex/drill.ts`), `summarize` вычитает проскок (~45%), обрезает паузы и латентность первого символа отрезка ещё **до** отправки. Из свёрнутых ячеек нельзя ни проверить альтернативное правило проскока, ни увидеть форму распределения, ни восстановить не сохранённую агрегацию. Первая когорта — единственная проверочная выборка фазы, а агрегат — дверь в одну сторону.

Решение: на время MVP включаем **сбор сырья** как **отдельный, параллельный, opt-out, time-boxed research-канал**. Клиент сериализует завершённый поток (`TypingRunRecord` — lossless, уже реализован в `src/lib/dev/typing-run.ts`), сжимает **`gzip`-потоком** (нативный `CompressionStream`, без зависимостей) и **best-effort** (at-most-once, ADR 0015) грузит **один блоб на сессию** в **Convex File Storage**; в таблицу `rawCaptures` пишется **тонкая строка метаданных** (для выбора когорты и удаления). Живой адаптивный цикл по-прежнему крутится на server-authoritative summary/профиле (ADR 0005) — сырьё его **не заменяет и не переопределяет**; из него агрегации выводятся/пересматриваются **офлайн**, в живой профиль переносятся только подтверждённые. Онлайн-аналитики по сырью нет by design: блоб непрозрачен, скачивается и разжимается пакетно; позже подметается на холодный бэкенд (R2/S3) по storage-URL'ам.

## Considered options

- **Только агрегации (статус-кво) — отвергнут.** Именно та задача, ради которой их собирают (калибровка чисел, проверка правил проскока/латентности/EWMA), из них **невыполнима**: они не сохраняют полноту и пред-фиксируют гипотезы под проверкой. Дёшево хранить, но данные первой когорты невосстановимы.
- **Сырьё attempts на сервер, как раньше запрещал ADR 0005 — контекст изменился.** Запрет 0005/0006 защищал *продуктовый путь* (тонкий клиент, профиль-контракт в символах, малый payload). Он остаётся в силе для живого цикла. Research-канал — **отдельная ось** (не профиль, не `drillRecord`), поэтому не отменяет 0005, а сужает его область: «сырьё не ходит на сервер **через профиль**».
- **Строка-на-нажатие в обычной таблице — отвергнут.** Взрыв строк/индексов (100 юзеров × десятки сессий × ~1–2k нажатий = миллионы строк), риск упереться в 0.5 ГБ БД. Сырьё не запрашивается онлайн → индексация бесполезна.
- **Инлайн `v.bytes()` в документе — допустим, но не выбран.** Проще (одна мутация), но смешивает тяжёлые байты с queryable-метой, ест квоту БД (0.5 ГБ) и не даёт URL для холодной выгрузки. **Выбрано: File Storage** (отдельная квота 1 ГБ, storage-URL, метаданные отдельно).
- **zip-архив — отвергнут.** Контейнер на много файлов; для одного блоба на сессию — лишняя обёртка. `gzip`-стрим проще и нативен.
- **opt-in vs opt-out.** На **бета-MVP выбран opt-out** (дефолт — собираем, лёгкий тумблер отказа): приглашённая когорта соглашается быть тестировщиками, печатается наш корпус (не приватный текст), чувствительность ограничена. Для будущего публичного режима планка иная — вернуться к opt-in (кандидат в отдельный ADR при выходе за пределы закрытой беты).

## Схема (черновик реализации, не часть решения)

```ts
// convex/schema.ts — добавить таблицу. Числа-тюнинга здесь не живут (политика README).
rawCaptures: defineTable({
  userId: v.id('users'),           // идентичность — только здесь; в блоб не попадает, при выгрузке аналитикам не отдаётся
  symbolLayoutId: v.string(),
  sessionId: v.optional(v.string()), // client-uuid: точное соединение raw ↔ sessionSummaries (nice-to-have)
  capturedAt: v.number(),          // server-stamped (как в sessionSummaries)
  openedSteps: v.number(),         // server-stamped: контекст когорты (ступень на момент записи)
  schemaVersion: v.number(),       // сырьё будет эволюционировать по мере пересмотра агрегаций
  symbolCount: v.number(),         // сколько StreamSymbol в потоке (быстрый отбор без скачивания блоба)
  uncompressedBytes: v.number(),
  compressedBytes: v.number(),
  blob: v.id('_storage'),          // gzip(JSON(TypingRunRecord)) в File Storage
}).index('by_user_and_layout', ['userId', 'symbolLayoutId']),
// _creationTime даёт хронологию (time-boxed отбор когорты).

// userSettings — добавить поле (opt-out; отсутствие = собираем):
rawCaptureOptOut: v.optional(v.boolean()),
```

Клиент: `streamToRunRecord` (готов) → `JSON.stringify` → `CompressionStream('gzip')` → upload URL File Storage → мутация `recordRawCapture(meta + storageId)`. Захват на `sessionComplete` (перехватчик `typing-capture.ts` переносится из dev-only в consented-путь). Плюс **серверный kill-switch** (env-флаг), чтобы выключить сбор, когда числа откалиброваны.

## Consequences

- **Поправляет ADR 0005** (при принятии): сырьё **может** уходить на сервер — но отдельным research-каналом, не через профиль/`drillRecord`. Продуктовый путь (тонкий клиент, summary-контракт) неизменен. При ратификации: статус-строка 0005 → «поправлен 0019», строка в индексе README.
- **Приватность — обязательства, без них не включать.** Раскрытие в политике приватности + лёгкий opt-out; удаление аккаунта (P0-4 плана запуска) **обязано** чистить и строку метаданных `rawCaptures`, и блоб в File Storage; time-boxed retention (удалять сырьё после исследовательского окна — data-minimization). Смягчает: печатается корпус, а не приватный текст; блоб обезличен.
- **Стоимость хранилища** — при `gzip` + один блоб на сессию комфортно под квоты free-tier; naive-строки-на-нажатие сорвали бы БД (почему и File Storage + компрессия).
- **Хвост сопровождения** — `schemaVersion` обязателен: сырьё будет меняться при пересмотре агрегаций; офлайн-анализ держит несколько версий. Онлайн-аналитики по сырью нет by design.
- **Не источник истины для UI/движка** — только офлайн-калибровка и golden-fixtures будущей `summarize` (ровно назначение из `typing-run.ts`). Живой цикл слепнет к сырью.
- Числа (окно retention, порог «числа откалиброваны») — не здесь (структура, не значения); дом — `config.ts`/`docs/plans/auto-flow.md`.
