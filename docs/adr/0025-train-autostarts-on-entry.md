# Вход на `/train` автоматически запускает тренировку; экран-меню упразднён

> Статус: принят · 2026-07-12 · Поправляет: ADR 0010 · Реализация: в коде

## Контекст

P0-11(a) вынес конфигурацию тренировки (язык текста, раскладки, курсор, ритм,
длительность) из экрана `menu` на `/settings`. После этого `menu` выродился в
**один экран с единственной кнопкой «Начать тренировку»**.

Но штатный вход на `/train` — это CTA «Начать тренировку» с лендинга
(`href="/train"`). Пользователь **уже нажал** эту кнопку — и на `/train` упирается
в такую же кнопку на пустом экране, которую надо нажать снова. Двойной клик по
идентичной кнопке = остаточное трение ровно там, где P0-11 обещал его убрать.

**ADR 0010 сам отложил это решение.** Он рассматривал прямой автостарт и отклонил
его — но по единственной причине: тогда `menu` был **экраном выбора**
(язык/раскладка/курсор), и пропуск его терял конфигурацию. Дословно: «menu — это
экран выбора… его пропуск — отдельное UX-решение, не часть этого фикса». P0-11(a)
убрал выбор из `menu` → причина отказа отпала, и «отдельное UX-решение на потом» —
это ровно то, что делает данный ADR.

## Решение — вход на `/train` автоматически запускает; `menu`-экрана нет

Авторизованный вход на `/train` **сразу запускает тренировку**, без промежуточного
экрана. `/train` = тренировка.

- **Механика.** `App.svelte` при входе на маршрут шлёт `TRAINER_OPENED` с
  `{ symbolLayoutId, durationSeconds }` из `$settings`; корневой обработчик
  `appMachine` переводит в `trainingStart` (→ `training`). Единый шов входа —
  наследие ADR 0010 — сохранён; изменилась лишь цель: `training` вместо `menu`.
- **`menu` → `idle`.** Бывшее `menu` переименовано в `idle`: внутреннее состояние
  «тренажёр в покое», **которое никогда не рисуется**. Его роль — цель гашения
  сессии при уходе с `/train` (`TRAINER_CLOSED → idle` завершает
  invoked-`sessionService` вместе с его таймером). Пользовательского экрана-меню нет.
- **Ядро ADR 0010 сохранено.** Причина 0010 — «вход не должен воскрешать залипший
  `sessionComplete`/брошенную сессию» — выполняется даже строже: автостарт даёт
  **свежую** сессию из любого прежнего состояния, а не показывает старый экран.
- **ADR 0007 (таймер) не тронут.** Автостарт лишь запускает `START_TRAINING`
  раньше (на входе, не по кнопке); сессия всё так же проходит `loading → armed →
  timing`, таймер стартует с первого нажатия. Семантика таймера неизменна.
- **Login wall (ADR 0012) цел.** Барьер — на уровне роута (`train/+page.svelte`:
  гость видит `SignInScreen`, `App` не смонтирован). Автостарт — только для
  авторизованного, в том числе сразу после входа через Google-popup (гейт
  становится тренажёром без навигации).
- **«В меню» упразднён** (некуда идти). После сессии `sessionComplete` даёт
  «Начать заново»; уход — через шапку (лого → `/`, Stats, Settings), что и гасит
  сессию. Пауза даёт «Продолжить» / «Начать заново». Событие `TO_MENU` и
  уведомление `MENU_START_REQUESTED` (Enter в меню) сняты как мёртвые.

## Рассмотренные альтернативы

- **Оставить `menu` как явную «точку покоя»** (автоматически запускает только вход, но «В
  меню» из паузы/результатов ведёт на тихий экран с CTA). Отвергнуто: сохраняет ту
  самую одинокую кнопку, на которую жалоба, — просто уводит её с входа. Шапка и так
  даёт все пути «уйти», отдельная точка покоя с кнопкой не нужна.
- **Автостарт только на первом забеге + определение первого запуска.** Отвергнуто ещё в
  проработке P0-11: сложнее (определение + расщепление поведения), а одиночный
  автостарт-на-входе одинаково хорошо обслуживает и первый забег, и возврат —
  оба суть «я пришёл тренировать».

## Последствия

- Возврат на `/train` всегда = свежая тренировка (взведённая, ждёт первого
  нажатия), а не экран-меню и не залипший прошлый экран. Канонические числа прошлой
  сессии остаются в `/stats` (их пишет `sessionMachine` на `done`).
- Шапка — единственный путь «уйти» с тренировки; уход гасит сессию
  (`TRAINER_CLOSED → idle`).
- `src/components/train/MenuScreen.svelte` удалён (единственный потребитель —
  `MainContent` — больше его не рендерит). i18n-ключ `app.back_to_menu` осиротел →
  прореживание в backlog.
- `TRAINER_OPENED` теперь несёт `{ symbolLayoutId, durationSeconds }`; `TO_MENU` и
  `MENU_START_REQUESTED` удалены из контракта событий. Обновлён `app.machine.test.ts`.
- ADR 0010 помечен «поправлен 0025» в статус-строке; его тело неизменно (политика
  `docs/adr/README.md`).
