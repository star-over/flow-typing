# Дизайн: `ui/MessageScreen`

Дата: 2026-07-21 · Ветка: `fix/css-focus-a11y-token-cleanup`

## Контекст

Аудит CSS выявил дублирование блоков (грань #3). В трёх местах рендерится
дословно одинаковый экран-сообщение: центрированная колонка, заголовок, текст,
одна ghost-кнопка.

- `src/components/device/KeyboardRequiredNotice.svelte` — мягкий гейт для
  тач-трафика (`.keyboard-required` + `__title` / `__body` / `__action`).
- `src/routes/+error.svelte` — страница ошибки навигации SvelteKit
  (`.error-page` + `__title` / `__body` / `__status` / `__action`).
- `src/routes/+layout.svelte` — `{#snippet failed()}` у `<svelte:boundary>`
  (`.app-error` + `__title` / `__body` / `__action`).

Кнопки идентичны байт-в-байт: десять свойств плюс `:hover` и `:focus-visible`.
Контейнеры совпадают во всём, кроме `margin`. Единственное содержательное
отличие — `+error` дополнительно печатает статус-код HTTP.

Существенно: ghost-кнопка нигде в проекте не встречается **вне** такого экрана.
Поэтому отдельный примитив кнопки для неё не нужен — один компонент закрывает и
семейство кнопок, и семейство экранов.

## Решение

Ввести `src/components/ui/MessageScreen.svelte` — доменно-нейтральный примитив
по действующей конвенции `src/components/CLAUDE.md` («`ui/` — generic-примитивы»).

### Контракт

```ts
const {
  title,
  body,
  note,
  actionLabel,
  onAction,
}: {
  title: string;
  body: string;
  note?: string;
  actionLabel: string;
  onAction: () => void;
} = $props();
```

Действие **обязательно**. Все три вызова имеют ровно одну кнопку; union-тип
«с действием / без» (идиома `KeyHint`) заводил бы ось под несуществующий вызов.

Все строки приходят пропсами уже локализованными (ADR 0022). `note` — строка,
не число: приведение (`String(page.status)`) выполняет вызывающий код.

### Разметка

Корневой `div` → `h1` (title) → `p` (body) → `p` (note, под `{#if}`) → `button`.

### Стили

Переносятся дословно из текущих трёх копий, с двумя правками:

1. **`margin` не переносится.** Все три экрана рендерятся внутри
   `<main class="main">` (`src/routes/+layout.svelte`), у которого
   `display:flex; flex-direction:column; justify-content:center; align-items:center`.
   Для флекс-элемента там `margin:auto` и `margin:4rem auto` дают тот же
   результат, что и родительское центрирование: горизонталь съедают `auto` и
   `align-items`, а `4rem` сверху и снизу симметричны и центр не смещают.
   Расхождение `4rem auto` в `+error` против `auto` в двух других —
   дрейф, не намерение.
2. **`font-variant-numeric: tabular-nums`** переезжает с `.error-page__status`
   на `.message-screen__note`. Для нецифрового текста это no-op, для статус-кода
   обязательно.

Геометрия — литералами и примитивами (`--spacing-*`, `--radius-3`,
`--font-size-*`, `--focus-ring-*`); цвета — через роли `var(--color-*)`
(ADR 0029). Новых ролей не заводится.

## Применение

| Место | Изменение |
| --- | --- |
| `src/routes/+layout.svelte` | `{#snippet failed()}` → `<MessageScreen>`; удаляются правила `.app-error*` |
| `src/routes/+error.svelte` | → `<MessageScreen note={...}>`; удаляются правила `.error-page*` |
| `src/routes/train/+page.svelte` | → `<MessageScreen>` напрямую; сюда переезжает комментарий про мягкий гейт |
| `src/components/device/` | удаляется целиком: `KeyboardRequiredNotice.svelte` и `KeyboardRequiredNotice.stories.svelte` |

`KeyboardRequiredNotice` после рефакторинга стал бы сквозным пробросом четырёх
пропсов без собственного содержания — обёртка удаляется, вызов идёт напрямую.
Это оставляет все три места однородными.

## Проверка

Компонент — dumb UI, автоматических тестов не получает: в `src/components/**` намеренно
ноль `*.test.ts` (ADR 0013). Проверка — витрина.

Новый файл `src/components/ui/MessageScreen.stories.svelte` (duo-конвенция),
истории:

- **With note** — статус-код присутствует.
- **Without note** — базовый случай.
- **Long text** — проверка `max-width: 28rem` и переноса заголовка.

Проверки: `make check-dev`, затем `make spell` и `make build` перед коммитом
(`make check-all`). Storybook — визуальная сверка с текущим видом.

## Что решено НЕ делать

- **ADR не заводится.** Добавление примитива в `ui/` применяет уже
  зафиксированную конвенцию (`src/components/CLAUDE.md`), а не меняет её; ADR по
  политике проекта оформляет смену курса. ADR станет уместен на следующем шаге
  грани #3 — `ui/Button` с системой вариантов (neutral / ghost / primary): это
  структурное решение о дизайн-системе, которого в конвенциях нет. Его
  предлагается оформить тогда, с опытом от `MessageScreen`.
- **`MainContent .session-error` в охват не входит.** Это вырожденный случай:
  `h2` вместо `h1` (не корневой заголовок — рендерится внутри тренировочного
  экрана), без кнопки, без `margin`. Втягивание добавило бы компоненту три
  опциональные оси ради одного вызова, а класс `.screen-title` всё равно остался
  бы в `MainContent` ради ветки «пауза».
- **Отдельный `ui/Button` в этой итерации не вводится** — см. выше про ghost.
- **`{#snippet}` в контракте не используется.** Контракт закрытый: компонент владеет
  разметкой и стилями целиком, вызывающий передаёт только данные.

## Область неопределённости

Утверждение о равнозначности `margin` выведено из разметки, а не измерено.
Сверить на витрине и в собранном приложении: если экран `+error` визуально
сместится, вернуть отступ — но параметром компонента, а не молча внутри него.
