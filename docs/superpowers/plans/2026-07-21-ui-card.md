# ui/Card Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ввести примитив `src/components/ui/Card.svelte` (оформление контент-карточки) и перевести три карточки на него.

**Architecture:** Полиморфный контейнер (`<svelte:element this={as}>`, `as` = `div`|`section`) с осью `padding` (none|sm|md) и «оформлением» `surface + 1px border + radius-4`. Содержимое — сниппет; per-site layout — у вызывающего через `{...rest}`.

**Tech Stack:** SvelteKit + Svelte 5 (runes), TypeScript strict, Storybook (svelte-csf), Tailwind v4 + shadcn-svelte на хроме (ADR 0035).

**Спек:** `docs/superpowers/specs/2026-07-21-ui-card-design.md` — источник истины.

## Global Constraints

- **Svelte 5 runes** — `$props()` с деструктуризацией; типы инлайн.
- **Цвет — только роли** `var(--color-*)` (ADR 0029); ни одного цветового литерала, кроме `transparent`.
- **Геометрия** — примитивы (`--spacing-*`, `--radius-*`) или литералы; не цвет-токены.
- **Type safety** — без `any`/escape; rest-пропсы типизированы `HTMLAttributes<HTMLElement>` из `svelte/elements`.
- **i18n (ADR 0022)** — ни одного строкового литерала-надписи в разметке компонентов/роутов. Исключение: `*.stories.svelte` (демо-данные).
- **Нет тестов компонентов (ADR 0013)** — проверка `svelte-check` + Storybook; jsdom/browser-mode/play-функции не вводить.
- **Импорты** — алиас `@/`.
- **Проверка задачи:** `make check-dev` (eslint + svelte-check + vitest) = 0 ошибок / 0 предупреждений. Перед коммитом — `make spell` (0 issues); перед финалом ветки — `make check-all`.
- **Ветка:** `feat/ui-card` (уже создана).

### Task 1: Компонент ui/Card + витрина

**Files:**
- Create: `src/components/ui/Card.svelte`
- Create: `src/components/ui/Card.stories.svelte`

**Interfaces:**
- Produces: `Card` (default export). Пропсы: `as?: 'div' | 'section'` (= `'div'`), `padding?: 'none' | 'sm' | 'md'` (= `'md'`), `children: Snippet`, плюс `...rest` (`HTMLAttributes<HTMLElement>` — `style`/`class`/`aria-*`/`id`/`data-*`). Рендерит `<div>` или `<section>` с оформлением `surface + border + radius-4`; `padding` → none(0)/sm(spacing-4)/md(spacing-6).

- [ ] **Step 1: Написать витрину (визуальный «тест»)**

Создать `src/components/ui/Card.stories.svelte`:

```svelte
<script module lang="ts">
  import { defineMeta } from '@storybook/addon-svelte-csf';
  import Card from './Card.svelte';

  const { Story } = defineMeta({
    title: 'ui/Card',
    component: Card,
  });
</script>

<Story name="Padding md (default)">
  <Card><p>Card content.</p></Card>
</Story>
<Story name="Padding sm">
  <Card padding="sm"><p>Card content.</p></Card>
</Story>
<Story name="Padding none">
  <Card padding="none"><p style="padding: 1rem;">Self-padded content.</p></Card>
</Story>
<Story name="As section">
  <Card as="section" aria-label="Demo region"><p>Rendered as section.</p></Card>
</Story>
<Story name="Constrained width">
  <Card style="max-width: 20rem;"><p>Width limited via caller style.</p></Card>
</Story>
```

- [ ] **Step 2: Убедиться, что витрина «падает» (компонента нет)**

Run: `make check`
Expected: FAIL — `svelte-check` не находит `./Card.svelte`.

- [ ] **Step 3: Реализовать компонент**

Создать `src/components/ui/Card.svelte`:

```svelte
<script lang="ts">
  import type { Snippet } from 'svelte';
  import type { HTMLAttributes } from 'svelte/elements';

  const {
    as = 'div',
    padding = 'md',
    children,
    ...rest
  }: HTMLAttributes<HTMLElement> & {
    as?: 'div' | 'section';
    padding?: 'none' | 'sm' | 'md';
    children: Snippet;
  } = $props();
</script>

<svelte:element
  this={as}
  {...rest}
  class="card"
  class:pad-sm={padding === 'sm'}
  class:pad-md={padding === 'md'}
>
  {@render children()}
</svelte:element>

<style>
  .card {
    background: var(--color-surface);
    border: 1px solid var(--color-border);
    border-radius: var(--radius-4);
  }
  .pad-sm { padding: var(--spacing-4); }
  .pad-md { padding: var(--spacing-6); }
</style>
```

Замечания: `class="card"` идёт ПОСЛЕ `{...rest}` — компонент владеет своим классом. `padding='none'` → ни `pad-sm`, ни `pad-md` → без padding. Директивы `class:` (не интерполяция) — чтобы Svelte видел `.pad-*` использованными.

- [ ] **Step 4: Проверить типы/рендер**

Run: `make check`
Expected: PASS — `0 errors, 0 warnings` (в т.ч. нет unused-selector). Если `svelte-check` ругается на `{...rest}`/`<svelte:element>`-типизацию — устранить БЕЗ смены API (не расширять `as`, не менять пропсы); если не выходит — сообщить BLOCKED с точной ошибкой. Затем `make storybook`, открыть `ui/Card` — три padding'а, `section`-рендер, ограничение ширины.

- [ ] **Step 5: Commit**

```bash
git add src/components/ui/Card.svelte src/components/ui/Card.stories.svelte
git commit -m "feat(ui): add Card primitive (surface chrome container)"
```
