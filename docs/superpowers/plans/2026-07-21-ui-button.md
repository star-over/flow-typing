# ui/Button Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ввести примитив `src/components/ui/Button.svelte` с системой вариантов и мигрировать на него семь дублирующихся кнопочных мест (грань #3 CSS-аудита).

**Architecture:** Полиморфный компонент (`<svelte:element this={href ? 'a' : 'button'}>`) с шестью семантическими вариантами и осью размера. Стили — scoped, цвета через роли `var(--color-*)`, геометрия — примитивы/литералы. Контент — сниппет `children` (композиция `KeyHint`). Layout — забота вызывающего.

**Tech Stack:** SvelteKit + Svelte 5 (runes), TypeScript strict, Storybook (svelte-csf). Без CSS-фреймворков.

**Спек:** `docs/superpowers/specs/2026-07-21-ui-button-design.md` — источник истины по решениям.

## Global Constraints

- **Svelte 5 runes** — `$props()` с деструктуризацией; типы инлайн.
- **Цвет — только через роли** `var(--color-*)` (ADR 0029). Ни одного цветового литерала (кроме `transparent`).
- **Геометрия** — примитивы (`--spacing-*`, `--radius-*`, `--font-size-*`, `--font-weight-*`, `--focus-ring-*`) или литералы; не токены-цвета.
- **i18n (ADR 0022)** — ни одного строкового литерала-надписи в разметке компонентов/роутов; метки приходят пропсами/словарём. **Исключение:** в `*.stories.svelte` литералы разрешены как демо-данные.
- **Тестов у компонентов нет (ADR 0013)** — проверка через `svelte-check` + Storybook; jsdom/browser-mode/play-функции не вводить.
- **Импорты** — алиас `@/` (= `src/`).
- **Параметры функций** — 1 позиционный, 2+ одним object literal (для функций, которые объявляем сами).
- **Проверка каждой задачи:** `make check-dev` (eslint + svelte-check + vitest) зелёный. Перед финальным коммитом ветки — `make check-all` (+ spell + build).
- **Ветка:** `fix/css-focus-a11y-token-cleanup` (уже существует; коммитим в неё).

```markdown
# ui/Button — примитив кнопки и модель вариантов

> Статус: принят · 2026-07-21 · Реализация: в работе (грань #3 CSS-аудита)

## Context

Аудит CSS (грань #3) нашёл дублирование кнопочных стилей: один визуальный
словарь (`surface + border`, плотная заливка, ghost-контур, brand-accent)
скопирован в независимые блоки `FooterActions` / `SettingsPage` / `stats` /
`SurveyPrompt` / `MessageScreen` / `LandingScreen` / `SignInScreen`. `MessageScreen`
(ADR-less примитив) уже вынес экран-сообщение, но ghost-кнопку держит вшитой.

Введение примитива кнопки с **системой вариантов** — это не применение
существующей конвенции (как было с `MessageScreen`), а новое структурное решение
о дизайн-системе: таксономия вариантов, способ полиморфизма элемента, где
проходит граница «стиль компонента ↔ лейаут владельца». Спек `MessageScreen`
явно отложил это решение сюда.

## Decision

1. **`src/components/ui/Button.svelte`** — доменно-нейтральный интерактивный
   примитив, покрывает только интерактивные контролы.
2. **Полиморфизм элемента:** проп `href` задан → `<a href>` (навигация); иначе →
   `<button type>` (действие). Различие «навигация ↔ действие» — реальное
   (доступность, новая вкладка, preload, семантика), моделируется явно.
3. **Шесть вариантов, семантические имена в одном пропе** `variant`:
   `neutral` / `primary` / `success` / `danger` / `ghost` / `accent`. Матрица
   `style × tone` отвергнута (даёт несуществующие комбинации).
4. **Foreground плотных вариантов** (`primary`/`success`/`danger`/`accent`)
   стандартизуется на `--color-on-dense` (роль «текст на плотной заливке»),
   устраняя дрейф `--color-background` в `FooterActions`.
5. **Ось размера** `size` (`md` / `lg`), отдельная от цвета.
6. **Граница стиль ↔ лейаут:** компонент владеет внутренним видом (padding,
   radius, цвета, состояния, `inline-flex`+gap); позиционирование (margin,
   align, width, flex) — у родителя. В Svelte scoped-стиль родителя до корня
   дочернего компонента не дотягивается, поэтому граница ещё и техническая.
7. **Контент — сниппет `children`** (композиция `KeyHint`), не закрытый контракт.

## Considered options

- **Готовая библиотека (shadcn/аналоги)** — отвергнута: shadcn = React+Tailwind,
  назван в запрете `CLAUDE.md`; Svelte-порт тоже на Tailwind. Развернул бы
  решение «CSS без фреймворков», притащил второй тематический движок, и не закрыл
  бы доменные вопросы (CTA-ссылка, KeyHint, i18n, темы).
- **Button-only без полиморфизма** — отвергнут: свёл бы CTA к кнопке, потеряв
  свойства ссылки на главной точке входа лендинга.
- **Матрица `style × tone`** — отвергнута в пользу семантических имён (YAGNI).
- **Втягивание сегмента/каретки** — отвергнуто: сегмент = будущий
  `SegmentedControl`, каретка = глиф; делят с кнопкой только токен цвета.

## Consequences

- Появляется общий словарь кнопок; семь мест перестают дублировать стили.
- `MessageScreen` теряет вшитую ghost-кнопку — переходит на `Button variant="ghost"`.
- Foreground плотных кнопок футера получает крошечный сдвиг тона (осознанно).
- `ui/Surface` (плашки) и `ui/SegmentedControl` (сегмент) — отдельные будущие
  примитивы; Button делит с ними роль-токены, но не код.
```

<script lang="ts">
  import type { Snippet } from 'svelte';

  const {
    variant = 'neutral',
    size = 'md',
    href,
    type = 'button',
    disabled = false,
    children,
    ...rest
  }: {
    variant?: 'neutral' | 'primary' | 'success' | 'danger' | 'ghost' | 'accent';
    size?: 'md' | 'lg';
    href?: string;
    type?: 'button' | 'submit';
    disabled?: boolean;
    children: Snippet;
  } = $props();
</script>

<svelte:element
  this={href ? 'a' : 'button'}
  {...rest}
  class="button {variant} {size}"
  {href}
  type={href ? undefined : type}
  disabled={href ? undefined : disabled}
>
  {@render children()}
</svelte:element>

<style>
  .button {
    display: inline-flex;
    align-items: center;
    gap: var(--spacing-2);
    border-radius: var(--radius-3);
    border: 1px solid transparent;
    font-family: var(--font-sans);
    cursor: pointer;
    text-decoration: none;
  }

  .button:disabled {
    opacity: 0.6;
    cursor: not-allowed;
  }

  /* --- sizes --- */
  .md {
    padding: var(--spacing-2) var(--spacing-4);
    font-size: var(--font-size-sm);
    font-weight: var(--font-weight-medium);
  }
  .lg {
    padding: var(--spacing-3) var(--spacing-6);
    font-size: var(--font-size-md);
    font-weight: var(--font-weight-semibold);
  }

  /* --- variants --- */
  .neutral {
    background: var(--color-surface);
    color: var(--color-text-primary);
    border-color: var(--color-border);
  }
  .neutral:hover:not(:disabled) {
    background: var(--color-surface-hover);
  }

  .primary {
    background: var(--color-primary-background);
    color: var(--color-on-dense);
    border-color: var(--color-primary-background);
  }

    background: var(--color-primary-hover);
    border-color: var(--color-primary-hover);
  }

  .success {
    background: var(--color-success);

    border-color: var(--color-success);
  }
  .success:hover:not(:disabled) {
    background: var(--color-success-hover);
    border-color: var(--color-success-hover);

    border-color: var(--color-error-hover);

    border-color: var(--color-text-secondary);
  }

    border-color: var(--color-text-primary);
  }

  .accent {
