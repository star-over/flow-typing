# Соглашения по именованию типов в TypeScript

Этот документ описывает правила именования типов в проекте для обеспечения консистентности и читаемости кода.

## 1. Общие правила

- **PascalCase**: Все типы, интерфейсы и псевдонимы типов (`type`) именуются в `PascalCase`.
- **Описательность**: Имена должны быть ясными, описательными и избегать неоднозначных сокращений.

## 2. Типы-объединения (Union Types)

Имя для объединения литералов (строк, чисел) должно быть в **единственном числе** и описывать *категорию* значений.

**Правильно:**
```typescript
// Описывает одну из нескольких возможных групп цвета
type KeyCapColorGroup = "PRIMARY" | "SECONDARY" | "ACCENT";

// Описывает один палец
type Finger = "L4" | "L3" | "L2" | "L1" | "L0" | "LB" | "RB" | "R0" | "R1" | "R2" | "R3" | "R4";
```

**Неправильно:**
```typescript
// Имя во множественном числе для типа, представляющего одно значение
type Fingers = "L4" | "L3" | ...;
```

## 3. Типы-объекты (Object Types)

Имя для типа, описывающего структуру объекта, должно быть существительным в **единственном числе**.

**Правильно:**
```typescript
// Описывает одну клавишу на физической клавиатуре
type PhysicalKey = {
  keyCapId: KeyEventCodes;
  unitWidth?: KeyCapUnitWidth;
};
```

**Неправильно:**
```typescript
// Суффикс "Item" является избыточным
type PhysicalKeyItem = { ... };
```

## 4. Типы-массивы (Array Types)

Подход к именованию зависит от структуры и назначения массива.

### 4.1. Простой массив (`T[]`)

Если для простого списка элементов создается отдельный тип, его имя должно быть во **множественном числе** от типа элемента.

**Пример:**
```typescript
type FingerZones = FingerZone[];
```
Однако в большинстве случаев предпочтительнее использовать `T[]` напрямую без создания отдельного типа-псевдонима.

### 4.2. Сложная структура или концептуальная группа

Если массив представляет собой сложную структуру (например, `T[][]`) или единую концепцию (например, раскладку), ему следует дать уникальное описательное имя в **единственном числе**.

**Пример:**
```typescript
// Описывает всю раскладку клавиатуры как единую сущность
type PhysicalKeyboardLayout = PhysicalKey[][];

// Описывает всю раскладку символов как единую систему
type SymbolLayout = SymbolMapping[];
```
Здесь `SymbolLayout` и `PhysicalKeyboardLayout` — это имена, описывающие целостную концепцию, а не просто "список".
