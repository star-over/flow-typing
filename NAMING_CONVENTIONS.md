# Соглашения по именованию типов в TypeScript

Этот документ описывает правила именования типов в проекте для обеспечения консистентности и читаемости кода.

## 1. Общие правила

- **PascalCase**: Все типы, интерфейсы и псевдонимы типов (`type`) именуются в `PascalCase`.
- **Описательность**: Имена должны быть ясными, описательными и избегать неоднозначных сокращений.

## 2. Типы-идентификаторы и Объединения (ID Types and Union Types)

Имя для объединения строковых или числовых литералов, представляющих идентификатор или категорию значений, должно быть в **единственном числе**.

**Правильно:**
```typescript
// Описывает одну из нескольких возможных групп цвета
type KeyCapColorGroup = "PRIMARY" | "SECONDARY" | "ACCENT";

// Описывает идентификатор пальца
type FingerId = "L4" | "L3" | "L2" | "L1" | "L0" | "LB" | "RB" | "R0" | "R1" | "R2" | "R3" | "R4";

// Описывает сторону руки
type HandSide = "Left" | "Right";
```

**Неправильно:**
```typescript
// Имя во множественном числе для типа, представляющего одно значение
type Fingers = "L4" | "L3" | ...;
```

## 3. Типы-объекты (Object Types)

Имя для типа, описывающего структуру объекта, должно быть существительным в **единственном числе**.

**Правильно:**
```typescript
// Описывает одну физическую клавишу (ее геометрию)
type PhysicalKey = {
  keyCapId: KeyCapId;
  unitWidth?: KeyCapUnitWidth;
};

// Описывает одну виртуальную клавишу для UI
type VirtualKey = {
  // ... объединенные данные и UI-состояние
};
```

**Неправильно:**
```typescript
// Суффикс "Item" является избыточным
type PhysicalKeyItem = { ... };
```

## 4. Типы-коллекции и Макеты (Collection and Layout Types)

Подход к именованию зависит от структуры и назначения коллекции.

### 4.1. Простой массив (`T[]`)

Если для простого списка элементов создается отдельный тип, его имя должно быть во **множественном числе** от типа элемента, или использовать суффикс `Layout` для коллекций, представляющих собой макет.

**Пример:**
```typescript
// Макет пальцев - коллекция анонимных объектов
type FingerLayout = {
  keyCapId: KeyCapId;
  fingerId: FingerId;
  isHomeKey?: boolean;
}[];
```
Однако в большинстве случаев предпочтительнее использовать `T[]` напрямую без создания отдельного типа-псевдонима.

### 4.2. Сложная структура или концептуальная группа

Если коллекция представляет собой сложную структуру (например, `T[][]`) или единую концепцию (например, раскладку), ей следует дать уникальное описательное имя в **единственном числе** с суффиксом `Layout`.

**Пример:**
```typescript
// Описывает всю физическую раскладку клавиатуры как единую сущность
type KeyboardLayout = PhysicalKey[][];

// Описывает всю символьную раскладку как единую систему
type SymbolLayout = {
  symbol: string;
  keyCaps: KeyCapId[];
}[];

// Описывает всю виртуальную клавиатуру для UI как единую сущность
type VirtualLayout = VirtualKey[][];
```
Здесь `KeyboardLayout`, `SymbolLayout` и `VirtualLayout` — это имена, описывающие целостную концепцию.
