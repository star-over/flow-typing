# src/interfaces — единый язык (типы)

Здесь id-объединения (`KeyCapId`, `FingerId`, `SYMBOL_LAYOUT_IDS`, `FINGER_LAYOUT_IDS`), типы слоёв раскладок, zod-схемы реестра. Базовое (3 слоя раскладок, правило «JSDoc — часть единого языка, не удалять») — в корневом `CLAUDE.md`; header `types.ts` несёт то же правило in-file. Ниже — ловушки, за которыми иначе идёшь в ADR.

- **`keyCapId` — всегда позиция** (`KeyboardEvent.code`; `KeyQ` = место на железе, не буква). `key-cap-id.ts` (`isKnownKeyCapId`) — граница «системный ввод → домен»: отфильтровывает неизвестные `code`, чтобы не засоряли аккорд.
- **«Инвариант» физического слоя ≠ «ANSI навсегда».** В JSDoc `PhysicalLayout` «инвариант» = геометрия не зависит от run-time выбора пользователя (холст не едет при смене символьной раскладки). Слой **не единственный**: `ansi`/`iso`/… растут добавлением членов union; ISO вводит `IntlBackslash` (ADR 0016).
- **Новый язык той же письменности без диакритики** добавляется **только** символьным слоем + `textLanguage` + корпусом (украинский = новый символьный слой на ANSI; QWERTY/QWERTZ/AZERTY/ЙЦУКЕН — символьный слой, не физический) (ADR 0016).
- **Форма записи символьной раскладки `{symbol, keyCaps}` объявлена трижды намеренно** — `shared/symbol-layout.ts` (слабый `string[]`), `src/interfaces/types.ts` (сильный `KeyCapId[]` + JSDoc), `src/lib/layouts.ts` (zod-гейт). Три мастера, дрейф принят: **не унифицировать**; каждый вход в `src` перепроверяется zod (ADR 0014).
