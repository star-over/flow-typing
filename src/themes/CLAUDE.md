# Темы и компонентные контракты

Покрывает также `*.contract.ts` рядом с компонентами. Полный гид и алгоритм добавления компонента — `docs/06-component-contracts-and-themes.md`.

Каждый компонент с темизируемыми элементами имеет рядом `*.contract.ts` — массив имён CSS-токенов, которые компонент использует через `var()`. Имена — это **визуальные роли** (`--keycap-l2-background`, `--footer-actions-btn-success-border`, `--keycap-home-ring`), не цвет; значение каждого токена — **полное** CSS-свойство (`1px solid oklch(…)`, `0 0 0 0.25rem oklch(…)`), не только цвет.

Все компонентные контракты агрегируются в `src/themes/contract.ts → THEME_CONTRACT`. Контракт-тест `src/themes/contract.test.ts` enforce-ит, что каждая тема (`src/themes/<id>.css`) и `_template.css` декларируют каждый токен; значения свободны.

Темы в `src/themes/`:
- `light` / `sepia` (colorScheme=light), `dark` / `nord` (colorScheme=dark). Каталог — `THEMES` в `src/themes/registry.ts`.
- **Трёхслойная схема (решение 2026-07-17, эталон — `sepia.css`).** Тема делится на три секции, граница между ними — правило, а не оформление. **Слой читается из префикса имени для каждого токена** (инвариант, ADR 0029): голое имя = ядро (L1); `--color-*` = роль (L2); `--<компонент>-*` = контракт (L3). Индустриальный маппинг: ЯДРО = primitive/option tokens · РОЛИ = semantic tokens · КОНТРАКТ = component tokens. **Целевая модель — двухслойная** (ядро + роли); контракт-слой транзитный — растворится будущей миграцией, когда роли стабилизируются (ADR 0029).
  1. **Ядро** — приватная палитра темы. Имена и количество произвольны (в `sepia` — `--parchment`, `--ink`, `--amber`…), значения **только абсолютные**. Единственное место, где в теме появляется новый цвет. Имена цветовые и это честно: внутри темы они описывают содержимое, а не роль.
  2. **Роли** — семантика, **фиксированный словарь имён** (`--color-*`, общий для всех тем). Каждая роль — маппинг/калькуляция от ядра **или от другой роли** (`oklch(from …)` — тюнинг светлоты/хромы/оттенка); новых абсолютов нет (ADR 0029).
  3. **Контракт компонентов** — `var()` на роли + **структурные литералы** (геометрия: числа width/offset/blur/spread, `solid`/`dashed`, `inset`, единицы). Запрещён любой **цветовой** литерал и формула — цветовой слот только `var()` на роль (нужна калькуляция цвета — значит в слое 2 не хватает роли, пример: `--color-error-hover`); единственный whitelist — `transparent` (ADR 0029).
- **Суффикс-словарь ролей** (источник истины именования, ADR 0029): `-foreground` — текст/символ на плотной заливке; `-hover` — hover-состояние роли; `-dim` — цвет приглушённой полупрозрачной поверхностью (альфа над фоном); `-strong` / `-muted` — усиленная / приглушённая ступень; `-raised` — приподнятая поверхность; `-accent` — тёплый / акцентный вариант; `-border` — граница.
- **Правило именования (ADR 0029).** Роль именует **семантику** (`background` / `foreground`), контракт — **CSS-свойство или элемент** (`background` / `color` / `border` / `ring`); слой при этом несёт префикс. Аббревиатуры раскрыты: `bg`→`background`, `fg`→`foreground`.
- **Структурная миграция завершена:** все темы (`sepia`, `light`, `dark`, `nord`) и `_template.css` — на трёхслойной схеме, словарь ролей выровнен до 73 имён (`ROLE_DICTIONARY`), расхождения слоя 3 между темами сведены к нулю; форму слоёв держит `describe('layer discipline')` в `contract.test.ts`.
- **Визуальная модель ADR 0028 перенесена во все темы.** Все темы несут маршрут и цель по пальцу; оттенки пальцев едины (ADR 0031, база `sepia`: h72/200/250/295/340), тема выводит только светлоту/хрому и архитектуру маршрут/цель. На тёмных темах (`dark`/`nord`) архитектура инверсна — цель это светлая заливка с тёмным символом (устраняет прежнюю просадку контраста мид-индиго-цели). Числа — расчётом (охват sRGB, WCAG, ΔE), не на глаз.
- **Индикатор готовности слоя 2 достигнут** (различия L3 = 0): когда пер-темные цвета устоятся, следующая крупная работа — вынести слой 3 из тем в код и растворить контракт (ADR 0029).
- `_template.css` — скелет для новой темы: каждый токен задан как `unset`, нужно только заполнить.
- Bootstrap синхронный — inline-script в `src/app.html` выставляет `data-theme` до первой отрисовки сцены. View Transitions API даёт crossfade при смене темы (`src/themes/registry.ts → setTheme`).
