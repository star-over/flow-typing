# UI entry points (роуты)

Клиентский auth и layout-оркестрация settings-sync — `src/lib/CLAUDE.md`.

- Роуты (каталоги в `src/routes/`): `/` (лендинг-placeholder с CTA на `/train`), `/train` (auth-барьер ADR 0012: гостю — приглашение войти, авторизованному — автостарт тренировки на входе, ADR 0025, далее FSM-views `training` / `sessionComplete` / `sessionError`; экрана-меню нет), `/settings` (приложение: язык UI + тема + имя), `/stats` (журнал сеансов `sessionSummaries` + прогресс ступени; гостю — приглашение войти), `/signin` (auth UI).
- `src/routes/+layout.svelte` — размещает `appActor`, keyboard listener (`<svelte:window>` onkeydown/up/blur → `KEY_DOWN`/`KEY_UP`/`PAUSE`), theme effects и `Header` (nav-chrome с ссылками на `/settings` и `/stats`). В `handleKeyDown` диспетчер пользовательских действий (`src/lib/user-actions/dispatch.ts`) перехватывает и аккорды с Cmd/Ctrl/Alt, и голые навигационные клавиши (Escape/Enter) **до** `appActor.send`: при совпадении с открытым по `when` действием оно выполняется, при промахе клавиша гасится (аккорд и навигационная клавиша) — в `keyboardMachine` они не попадают (ADR 0032). Контекст диспетчера собирается здесь: `isActive` = `inState(appState)`, `send` = `appActor.send`, `navigate` = `goto(resolve(route))`, `trainingParams` из `$settings`. При sibling-навигации layout не размонтируется — FSM состояние переживает навигацию.
- `src/routes/+page.svelte` — лендинг с CTA «Начать тренировку» (`href="/train"`). Inline-placeholder, контракт темы не выделен (tech-debt note inline; запись в `docs/backlog.md`).
- `src/routes/train/+page.svelte` → `src/components/app/App.svelte` — содержимое `/train`; рендерит `MainContent` (выбор по `state.matches(...)`) + `FooterActions` (process-controls, виден только в `training`/`sessionComplete`/`sessionError`).
- `Space` в `training` блокируется (`preventDefault`), чтобы не прокручивать.
- `TrainingScene.svelte` получает `sessionActor` (`state.children.sessionService`), подписывается на него для таймера и строит вложенный `trainingActor` (`sessionState.children.training`); передаёт `viewModel` в `HandsScene.svelte`; `FlowLine` показывает поток символов с курсором.
