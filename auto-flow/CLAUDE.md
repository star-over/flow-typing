# auto-flow — мастерская Auto-Flow (tooling)

Инструменты **подготовки данных**: читают тексты/раскладки с диска, чистят и строят корпус drill'ов, запускаются отдельно (нативный TS, Node ≥ 22, **без tsc/dist**: `node auto-flow/scripts/<файл>.ts …`). Это **tooling**, не модель времени выполнения — та живёт в `shared/` (её импортируют и сервер, и мастерская). Граница слоёв: `auto-flow` **не импортирует** `convex`.

Зачем и каким языком — `CONTEXT.md` (глоссарий) и `docs/adr/`; что и в каком порядке строить, числа-настройки — `docs/plans/auto-flow.md`. Команды — в `Makefile` (`corpus-process` / `build-corpus` / `import-corpus` / `ladder-report`); флаги CLI — в `--help` скриптов.

### Где что живёт (карта границ)

- **Модель времени выполнения** (рост репертуара, расчёт `stepLevel`, чистые помощники раскладки) — `shared/`.
- **Данные раскладок** — `src/layouts/*.json` (единый источник: грузит приложение, читает мастерская, импортирует сервер).
- **Схема и серверные функции** (`drills` / `drillSelectionIndex`, выдача порции, серверный пересчёт таблицы отбора) — `convex/`.
- **Клиент тренировки** — `src/` (SvelteKit).
- **Данные корпуса** (`auto-flow/corpus/**`) — **gitignored** (большие, регенерируемые); в git только скрипты.

### Конвейер корпуса

Чистая функция `тексты + набор символов раскладки → записи drills + отчёт покрытия`. Раскладка — **параметр**: единица годится, только если все её символы входят в набор символов раскладки (**членство заменяет определение языка**).

Три ступени: `jokes-csv-to-text` (CSV шуток → чистый текст) → `clean-corpus` (нормализация + фильтры, промежуточный этап — `drills.jsonl` **не** создаёт) → `build-corpus` (**финал**: `drills.jsonl` + `meta` + отчёт покрытия символов). Оркестратор шагов — `pipeline.ts` (`buildDrills`), сами шаги (`clean` / `segment` / `filters` / `meta` / `coverage`) — чистые.

Инвариант: **дедупликация — всегда последний шаг** (после всей нормализации/фильтрации, для схлопывания вариантов, сведённых нормализацией к одинаковым).
