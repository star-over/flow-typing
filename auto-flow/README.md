# auto-flow/ — мастерская Auto-Flow (tooling)

Инструменты **подготовки данных** для адаптивной системы **Auto-Flow**: читают тексты/раскладки с диска, строят корпус, запускаются отдельно. Это **tooling**, не рантайм-модель — её место в [`shared/`](../shared).

Зачем и каким языком — [`CONTEXT.md`](../CONTEXT.md) (глоссарий) и [`docs/adr/`](../docs/adr). Что и в каком порядке строить — [`docs/plans/auto-flow.md`](../docs/plans/auto-flow.md).

## Структура

```
auto-flow/
├── corpus/                  конвейер корпуса (чистые шаги + тесты)
│   ├── pipeline.ts          clean → segment → фильтры → дедупликация → meta → coverage
│   ├── clean.ts segment.ts filters.ts meta.ts coverage.ts types.ts
│   └── corpus.test.ts
├── scripts/build-corpus.ts  оболочка: текст → конвейер → drills.jsonl + отчёт
├── symbol-layout.ts         fs-загрузчик раскладки (чистые помощники — в shared/)
├── string-normalization.ts  нормализация текста
├── data/                    тексты-источники и выгрузка корпуса — gitignored
└── README.md
```

## Где что живёт

- **Рантайм-модель** (KeyLadder, расчёт `stepLevel`, чистые помощники раскладки) — в [`shared/`](../shared); её импортируют и сервер (Convex), и эта мастерская.
- **Данные раскладок** — `src/data/layouts/*.json` (единый источник: грузит приложение, читает мастерская, импортирует сервер).
- **Схема и серверные функции** — `convex/` (таблицы `drills`/`drillSelectionIndex`, выдача порции, серверный пересчёт таблицы отбора).
- **Рантайм-клиент тренировки** — `src/` (SvelteKit).

## Конвейер корпуса

Чистая функция `тексты + набор символов раскладки → записи drills + отчёт покрытия`. Раскладка — **параметр**: единица годится, только если все её символы входят в набор символов раскладки (членство заменяет определение языка). Шаги — в заголовках файлов `corpus/`.

Запуск:
- **`make corpus-process`** — весь цикл одной командой: сборка → заливка → пересчёт таблицы отбора.
- По отдельности: `make build-corpus` (текст по строкам → конвейер → `data/drills.jsonl` + статистика и дыры покрытия) → `make import-corpus` (заливка в Convex `--replace` **и** пересчёт таблицы отбора — после смены `_id` это обязательно, поэтому объединено в одну цель).
