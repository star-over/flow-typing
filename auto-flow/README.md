# auto-flow/ — мастерская Auto-Flow (tooling)

Инструменты **подготовки данных** для адаптивной системы **Auto-Flow**: читают тексты/раскладки с диска, чистят и строят корпус drill'ов, запускаются отдельно. Это **tooling**, не модель времени выполнения — её место в [`shared/`](../shared).

Зачем и каким языком — [`CONTEXT.md`](../CONTEXT.md) (глоссарий) и [`docs/adr/`](../docs/adr). Что и в каком порядке строить — [`docs/plans/auto-flow.md`](../docs/plans/auto-flow.md).

## Структура

```
auto-flow/
├── scripts/                   весь код мастерской: CLI + модули конвейера + помощники + тесты
│   ├── jokes-csv-to-text.ts   CSV шуток → чистый текст (извлечь + фильтр + дедупликация)
│   ├── clean-corpus.ts        текст → очищенный текст (нормализация + фильтры + дедупликация)
│   ├── build-corpus.ts        очищенный текст → drills.jsonl + отчёт покрытия
│   ├── pipeline.ts            конвейер buildDrills: clean → segment → фильтры → дедупликация → meta → coverage
│   ├── clean.ts segment.ts filters.ts meta.ts coverage.ts types.ts   шаги конвейера (чистые)
│   ├── symbol-layout.ts       fs-загрузчик раскладки (чистые помощники — в shared/)
│   ├── string-normalization.ts  нормализация текста (+ .test.ts)
│   └── corpus.test.ts
├── corpus/                    источники и выгрузка корпуса — gitignored
│   └── en/
│       ├── raw/               источники: oxford/ (PDF), short-jokes-dataset/ (CSV)
│       ├── derived/           очищенный текст (*.txt)
│       └── jsonl/             готовые drills.jsonl для Convex
└── README.md
```

## Где что живёт

- **Модель времени выполнения** (KeyLadder, расчёт `stepLevel`, чистые помощники раскладки) — в [`shared/`](../shared); её импортируют и сервер (Convex), и эта мастерская.
- **Данные раскладок** — `src/data/layouts/*.json` (единый источник: грузит приложение, читает мастерская, импортирует сервер).
- **Схема и серверные функции** — `convex/` (таблицы `drills`/`drillSelectionIndex`, выдача порции, серверный пересчёт таблицы отбора).
- **Клиент времени выполнения тренировки** — `src/` (SvelteKit).
- **Данные корпуса** (`auto-flow/corpus/**`) — **gitignored** (большие, регенерируемые). В git только скрипты.

## Конвейер корпуса

Чистая функция `тексты + набор символов раскладки → записи drills + отчёт покрытия`. Раскладка — **параметр**: единица годится, только если все её символы входят в набор символов раскладки (членство заменяет определение языка).

```
raw CSV/PDF ──jokes-csv-to-text──▶ derived/*.txt ──clean-corpus──▶ очищенный *.txt ──build-corpus──▶ jsonl/*.jsonl
  (источник)     (извлечь + фильтр)     (текст)        (нормализация + дедупликация)     (текст)      (финал, под раскладку)
```

Скрипты — нативный TS (Node ≥ 22, без tsc/dist): `node auto-flow/scripts/<файл>.ts …`.

### 1. `jokes-csv-to-text.ts` — CSV шуток → чистый текст
Извлекает колонку `Joke` (без ID, шапки, CSV-обвеса), фильтрует и дедуплицирует.
```
node auto-flow/scripts/jokes-csv-to-text.ts --input <csv|каталог> [--output <каталог>] [--max-len 120]
```
- **Порядок:** извлечь → отбросить диалог/метки (`Q:`, `Teacher:`, `First witch:` — 1–3 слова + `:`, а также `Q.`/`A.`) → отбросить длинные (> `--max-len`, дефолт 120 символов) → **дедупликация (последним)**.
- `--input` — файл или каталог (тогда каждый `*.csv` → свой `.txt`). `--output` — каталог (дефолт `corpus/en/derived`), имя = `<basename>.txt`.

### 2. `clean-corpus.ts` — текст → очищенный текст (без drills.jsonl)
Прогоняет через штатный конвейер и пишет только очищенные строки.
```
node auto-flow/scripts/clean-corpus.ts --input <txt|каталог> [--layout qwerty] [--output <каталог>]
```
- **Порядок (`pipeline.ts`):** clean/нормализация → segment → фильтры (начало · длина 2–150 · доля букв ≥0.5 · членство в раскладке) → **дедупликация (последним)**.
- `--layout` — для фильтра «символ в раскладке» (дефолт `qwerty`). `--output` — дефолт **каталог входа = чистка НА МЕСТЕ** (перезаписывает `.txt`). Мета и `drills.jsonl` НЕ создаются — промежуточный этап.

### 3. `build-corpus.ts` — очищенный текст → `drills.jsonl` (ФИНАЛ)
Собирает единицы-drill'ы с метаданными под конкретную раскладку. Делать **последним**, когда всё почищено.
```
make build-corpus LAYOUT=qwerty INPUT=<txt|каталог> OUTPUT=<путь.jsonl>
# или: node auto-flow/scripts/build-corpus.ts --layout qwerty --input <…> --output <…>
```
- Тот же конвейер + `meta` (длина, биграммы, частоты…) + отчёт покрытия символов (дыры видны сразу).

### Модули конвейера (библиотека, не CLI)
`pipeline.ts` (оркестратор `buildDrills`) · `clean.ts` (нормализация) · `segment.ts` · `filters.ts` (предикаты качества) · `meta.ts` · `coverage.ts` · `types.ts`. Тесты — `corpus.test.ts`.

### Принцип: дедупликация — всегда последний шаг
После всей нормализации/фильтрации (варианты, сведённые нормализацией к одинаковым, дедуплицируются). Оба cleaning-скрипта завершаются дедупликацией.

## Заливка на сервер

- **`make corpus-process`** — весь цикл одной командой: сборка → заливка → пересчёт таблицы отбора.
- По отдельности: `make build-corpus` → `make import-corpus` (заливка в Convex `--replace` **и** пересчёт таблицы отбора — после смены `_id` это обязательно, поэтому объединено в одну цель).
- **`make ladder-report LAYOUT=<id>`** — проверка распределения корпуса по ступеням KeyLadder.
