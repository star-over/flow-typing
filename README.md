# FlowTyping: Приложение для тренировки слепой печати

Это приложение Next.js для тренировки слепой печати, разработанное с использованием TypeScript, React и Tailwind CSS.

## Философия проекта

**FlowTyping** — это не просто очередной клавиатурный тренажер. В основе проекта лежит философия **"рефлексивно-адаптивного обучения"**. Мы отказываемся от скучного заучивания клавиш в пользу формирования устойчивой мышечной памяти. Обучение строится на печати осмысленного текста, а не на монотонных упражнениях, что делает процесс увлекательным и эффективным.

Ключевая инновация — **"визуализация движения"**, которая учит руки двигаться правильно, а не просто показывает, куда нажать.

## Ключевые особенности

*   **Рефлексивно-адаптивное обучение:** Система нацелена на создание рефлексов "символ -> движение пальца", а не на сознательное запоминание.
*   **Визуализация движения:** Вместо статичной виртуальной клавиатуры, интерфейс подсказывает траекторию движения пальцев к целевой клавише.
*   **Практика на реальном тексте:** Обучение с самого начала проходит на реальных словах и предложениях.
*   **Интеллектуальный подбор упражнений:** Алгоритм анализирует ваши слабые места и формирует персонализированные сессии для их отработки.
*   **Строка потока (`FlowLine`):** Интерактивная строка ввода, имитирующая каретку печатной машинки для лучшей концентрации.

## Начало работы

### Предварительные требования

Убедитесь, что у вас установлен Node.js (рекомендуется версия 18.x или выше) и npm.

### Установка

1.  Клонируйте репозиторий:
    ```bash
    git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
    cd flow-typing
    ```
2.  Установите зависимости:
    ```bash
    npm install
    ```

### Запуск приложения

#### Режим разработки

Запустите сервер разработки Next.js:
```bash
make dev
```
Приложение будет доступно по адресу `http://localhost:3000`.

#### Storybook

Запустите Storybook для разработки и тестирования компонентов:
```bash
make storybook
```
Storybook будет доступен по адресу `http://localhost:6006`.

#### Сборка для продакшена

Создайте оптимизированную сборку приложения:
```bash
make build
```
Результаты сборки будут находиться в каталоге `.next`.

#### Тестирование

Запустите тесты с использованием Vitest:
```bash
make test
```

#### Проверка типов и линтинг

Для выполнения всех проверок используйте команду:
```bash
make check-all
```
Эта команда включает в себя проверку типов, линтинг, тестирование и сборку проекта.

## Структура проекта

*   `src/app/`: Основные файлы приложения Next.js (маршруты, глобальные стили).
*   `src/components/`: Компоненты пользовательского интерфейса, включая общие UI-компоненты (`src/components/ui`).
*   `src/data/`: Данные приложения, такие как раскладки клавиатуры, зоны пальцев и раскладки символов.
*   `src/interfaces/`: Определения типов TypeScript.
*   `src/lib/`: Вспомогательные утилиты и логика, например, алгоритм определения навигационных статусов клавиш.

## Соглашения по разработке

### Стиль кода

Проект использует стандартный стиль кода Next.js с ESLint для линтинга. Конфигурация находится в `eslint.config.mjs`.

### Соглашения по именованию

Подробные правила именования типов описаны в файле [NAMING_CONVENTIONS.md](./NAMING_CONVENTIONS.md).

### Компоненты

Компоненты UI используют `class-variance-authority` для создания вариантов.

### Данные

*   `symbol-layout-en-qwerty.ts`: Определяет символы для каждой клавиши.
*   `finger-layout-asdf.ts`: Сопоставляет клавиши с пальцами.
*   `keyboard-layout-ansi.ts`: Описывает физическую структуру клавиатуры.

### Алгоритм определения навигационных статусов клавиш

Для интерактивного обучения используется система навигационных статусов (`TARGET`, `PATH`, `NONE`), которая визуально подсказывает пользователю, куда двигать палец. Логика реализована в функции `findPath` в файле `src/lib/virtual-layout.ts`. Подробное описание алгоритма можно найти в `GEMINI.md`.
