# Приложение для тренировки слепой печати

Это приложение Next.js для тренировки слепой печати, разработанное с использованием TypeScript, React и Tailwind CSS. Оно предоставляет визуальное представление клавиатуры и помогает пользователям изучить правильное расположение пальцев для каждой клавиши.

## Ключевые особенности

*   **Визуальная клавиатура:** Интерактивное визуальное представление клавиатуры, обеспечивающее обратную связь при нажатии клавиш.
*   **Зоны пальцев:** Динамическая подсветка правильного пальца для каждой клавиши, помогающая освоить правильную постановку рук.
*   **Практика печати:** Пользователи могут практиковаться в печати, получая обратную связь о своей точности и скорости.

## Используемые технологии

*   **Фреймворк:** Next.js
*   **Язык:** TypeScript
*   **Библиотека UI:** React
*   **Стилизация:** Tailwind CSS
*   **Разработка компонентов:** Storybook
*   **Тестирование:** Vitest
*   **Линтинг:** ESLint

## Начало работы

### Предварительные требования

Убедитесь, что у вас установлен Node.js (рекомендуется версия 18.x или выше) и npm.

### Установка

1.  Клонируйте репозиторий:
    ```bash
    git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
    cd flow-typing
    ```
2.  Установите зависимости:
    ```bash
    npm install
    ```

### Запуск приложения

#### Режим разработки

Запустите сервер разработки Next.js:
```bash
npm run dev
```
Приложение будет доступно по адресу `http://localhost:3000`.

#### Storybook

Запустите Storybook для разработки и тестирования компонентов:
```bash
npm run storybook
```
Storybook будет доступен по адресу `http://localhost:6006`.

#### Сборка для продакшена

Создайте оптимизированную сборку приложения:
```bash
npm run build
```
Результаты сборки будут находиться в каталоге `.next`.

#### Тестирование

Запустите тесты с использованием Vitest:
```bash
npm run test
```

#### Проверка типов и линтинг

*   **Проверка типов:**
    ```bash
    npm run type-check
    ```
*   **Линтинг:**
    ```bash
    npm run lint
    ```

## Структура проекта

*   `src/app/`: Основные файлы приложения Next.js (маршруты, глобальные стили).
*   `src/components/`: Компоненты пользовательского интерфейса, включая общие UI-компоненты (`src/components/ui`).
*   `src/data/`: Данные приложения, такие как раскладки клавиатуры, зоны пальцев и раскладки символов.
*   `src/interfaces/`: Определения типов TypeScript.
*   `src/lib/`: Вспомогательные утилиты и логика, например, алгоритм определения навигационных статусов клавиш.

## Соглашения по разработке

### Стиль кода

Проект использует стандартный стиль кода Next.js с ESLint для линтинга. Конфигурация находится в `eslint.config.mjs`.

### Соглашения по именованию

Подробные правила именования типов описаны в файле [NAMING_CONVENTIONS.md](./NAMING_CONVENTIONS.md).

### Компоненты

Компоненты UI используют `class-variance-authority` для создания вариантов.

### Данные

*   `symbol-layout-en-qwerty.ts`: Определяет символы для каждой клавиши.
*   `finger-layout-asdf.ts`: Сопоставляет клавиши с пальцами.
*   `keyboard-layout-ansi.ts`: Описывает физическую структуру клавиатуры.

### Алгоритм определения навигационных статусов клавиш

Для интерактивного обучения используется система навигационных статусов (`HOME`, `TARGET`, `PATH`, `IDLE`), которая визуально подсказывает пользователю, куда двигать палец. Логика реализована в функции `findPath` в файле `src/lib/virtual-layout.ts`. Подробное описание алгоритма можно найти в `GEMINI.md`.
