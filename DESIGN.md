# Design System: FlowTyping

## 1. Overview

**Creative North Star: «Тихий метроном»**

FlowTyping выглядит как ровный, спокойный метроном: всё внимание — на движении и ритме, а сам инструмент молчит. Хром — почти отсутствующий: нейтральные оттенки серого, тонкие границы, никакого декора, который соревновался бы за взгляд. Насыщенность здесь — драгоценный ресурс, и она целиком отдана единственному, что важно: **визуализации движения пальцев** (спектр пальцев, путь, кольцо цели). Поверхность вокруг тренировки должна читаться как пустой стол под метрономом — чтобы стрелка была видна.

Система обслуживает задачу, а не себя. Это продуктовый регистр: фиксированная rem-шкала вместо текучего clamp, один шрифт-семейство для всего UI, предсказуемый словарь компонентов от экрана к экрану. Доверие к методике рождается из точности — поэтому интерфейс ведёт себя ровно так, как ожидаешь, и никогда не удивляет ради удивления. Тактильность зарезервирована за клавишами (они физически «нажимаются»), хром остаётся тихим.

Система явно **отвергает**: громкую геймификацию в духе Duolingo (маскоты, серии, конфетти на каждом шагу); стерильный корпоративный SaaS (navy + серый, безликие карточки, «ещё одна корпоративная панель»); и визуальный шум перегруженных конкурентов (typing.com / ratatype) с их busy-панелями и рекламой.

**Key Characteristics:**
- Нейтральные оттенки серого для хрома; вся насыщенность — на визуализации движения.
- Спектр из пяти постоянных «пальцевых» оттенков (большой → мизинец).
- Плоско по умолчанию: глубина через границы и кольца, не через тени.
- Geist для UI, Geist Mono для потока печати.
- Четыре темы (light / sepia / dark / nord) с одинаковыми ролями токенов.
- Тактильны только клавиши; окружающий UI сдержан.

## 2. Colors: Тихий нейтрал и спектр пальцев

Палитра делится на две вселенные: тихие нейтральные оттенки серого для всего хрома и поверхностей, и насыщенный спектр, который принадлежит **только** визуализации движения.

### Primary
- **Янтарь цели** (`oklch(0.77 0.16 70)`): главный акцент. Метка цели (target-marker), граница курсора, CTA лендинга. Появляется там, куда должен двигаться палец — и больше почти нигде.

### Secondary
- **Индиго пути** (`oklch(0.62 0.19 260)`): подсветка навигационного пути от дома к цели и центральная точка `HandsScene`. Холодный противовес тёплому янтарю цели.

### Tertiary — Спектр пальцев
Пять постоянных оттенков, по одному на палец каждой руки (L/R зеркальны). Это единственная сильно-насыщенная палитра системы, и она закреплена за клавишами и пальцами:
- **Большой — янтарь** (`oklch(0.86 0.17 92)`)
- **Указательный — оранжевый** (`oklch(0.76 0.16 56)`)
- **Средний — зелёный** (`oklch(0.80 0.18 152)`)
- **Безымянный — синий** (`oklch(0.71 0.14 255)`)
- **Мизинец — фиолетовый** (`oklch(0.72 0.18 306)`)

### Neutral
- **Чистый лист** (`oklch(1 0 0)`): фон документа (light). Пустой стол под метрономом.
- **Тихая поверхность** (`oklch(0.95 0 0)`): панели, кнопки, поля, граница.
- **Графит** (`oklch(0.14 0 0)`): основной текст; заливка primary-кнопки.
- **Вторичный / приглушённый текст** (`oklch(0.44 0 0)` / `oklch(0.56 0 0)`): подписи, единицы, второстепенное.

### Семантика исходов
- **Успех** (`oklch(0.72 0.19 150)`), **ошибка** (`oklch(0.64 0.21 25)`), **предупреждение** (`oklch(0.77 0.16 70)`): статусы кнопок процесса и результаты нажатий. Янтарь предупреждения совпадает с янтарём цели намеренно.

### Named Rules
**Правило тихого хрома.** Весь окружающий UI держится на нейтральных оттенках серого (chroma 0). Насыщенный цвет зарезервирован за визуализацией движения — кнопки, поля, навигация, заголовки цвет не несут. Если хром стал цветным, что-то пошло не так.

**Правило спектра пальцев.** Каждый палец имеет постоянный закреплённый оттенок во всех темах. Этот спектр нельзя переназначать, разбавлять или применять к не-клавишам: цвет здесь — это смысл (какой палец), а не украшение.

## 3. Typography

**UI Font:** Geist (с `system-ui, sans-serif`)
**Stream / Mono Font:** Geist Mono (с `monospace`)

**Character:** Geist — нейтральный геометрический гротеск без характера-выскочки: он не отвлекает. Печатаемый поток набран моноширинным Geist Mono, чтобы символы стояли в ровной сетке и «не прыгали» при наборе — визуальный аналог ровного ритма метронома. Шкала фиксированная (производна от `--font-size-base: 1rem` по множителям Tailwind v4), не текучая.

### Hierarchy
- **Title** (Geist, 600, до `1.5rem` / 2xl, lh 1.2): заголовки экранов и секций.
- **Body** (Geist, 400, `1rem`, lh 1.5): основной текст; проза держится в 65–75ch.
- **Label** (Geist, 500, `0.875rem`): подписи кнопок, полей, компонентов управления.
- **Stream** (Geist Mono, 400, `1.875rem`, lh 1.2): строка печатаемого потока (`FlowLine`) с курсором. Самый крупный типографический элемент на экране тренировки.
- **KeyCap label** (Geist, 200–900 по состоянию, `0.5625–0.8125rem`): надписи на клавишах; вес растёт со значимостью (target → 900, press-result → 800).

### Named Rules
**Правило моно-потока.** Печатаемый поток всегда моноширинный (Geist Mono). Пропорциональный шрифт в потоке запрещён — он ломает ровную сетку символов и ритм.

**Правило фиксированной шкалы.** В продуктовом UI — фиксированная rem-шкала, никаких `clamp()` / fluid-заголовков. Пользователь смотрит при постоянном DPI; «съезжающий» размер делает хуже, не лучше.

## 4. Elevation

Система **плоская по умолчанию**. Поверхности плоские в покое; ощущение глубины создаётся тонкими границами (1px) и `box-shadow`, который используется не как тень-подъём, а как **кольцо** вокруг клавиши (home-ring, path-ring, target-ring). Тени-примитивы в системе заданы (`--shadow-sm … --shadow-xl`, нейтральный Tailwind v4 set), но компоненты их сейчас намеренно не используют — это резерв, а не язык по умолчанию.

### Shadow Vocabulary (резерв)
- **`--shadow-sm`** (`0 1px 2px 0 rgb(0 0 0 / 0.05)`) … **`--shadow-xl`** (`0 20px 25px -5px …`): 4-ступенчатая нейтральная лестница. Семантика — высота над поверхностью. Держать в резерве для будущих overlay/dropdown, не рассыпать по плоским поверхностям.

### Named Rules
**Правило кольца, а не тени.** `box-shadow` на клавишах — это кольцо-обводка (`0 0 0 Npx color`), маркирующее роль (дом / путь / цель), а не размытая тень-подъём. Размытые drop-shadow на клавишах и панелях запрещены.

**Правило плоскости по умолчанию.** Поверхности плоские в покое. Если поверхности понадобилась тень — сначала спроси, не решается ли это границей или кольцом.

## 5. Components

### Buttons (FooterActions, SettingsPage)
- **Shape:** мягко скруглённые углы (6px, `--radius-3`).
- **Default:** тихая поверхность, графитовый текст, граница 1px (`8px 16px` padding, вес 500). Hover — чуть темнее поверхность.
- **Primary:** инверсия — графитовая заливка, светлый текст. Hover — `opacity: 0.9`.
- **Success / Warning / Danger:** семантическая заливка (зелёный / янтарь / красный) со светлым текстом. Применять только к элементам управления процессом (старт / пауза / завершение), не к навигации.
- **Transition:** `background-color 0.1s ease` — быстрая обратная связь, без хореографии.

### Select (Select.svelte)
- **Style:** тихая поверхность, граница 1px, скругление 6px; нативная стрелка убрана (`appearance: none`).
- **Arrow:** mask-стрелка (chevron), цвет тянется из токена `--select-arrow-background` и адаптируется к теме.
- **Focus:** видимый `outline` 2px с `outline-offset: 2px`.

### KeyCap (signature)
- **Shape:** 4px (`--radius-2`), высота `2rem`, ширина кратна `--keycap-unit: 2.25rem`.
- **Per-position fill:** каждая клавиша окрашена по своему пальцу (`data-finger-id="L1…R5"` → `--keycap-{pos}-background/border/color`).
- **States:** `home` (кольцо-обводка + маркер-точка/полоска), `role-path` (кольцо пути, вес 700), `role-target` (заливка цели + кольцо, вес 900), press-results `CORRECT` / `ERROR` (заливка результата поверх роли, кольцо цели остаётся, вес 800). Правило Полного Кластера: `keyCapStates` задаются только у пальцев в состоянии TARGET.
- **Tactile:** `:active { transform: translateY(1px) }` — клавиша физически «нажимается». Это единственное место, где UI имитирует физику.

### FlowLine (signature)
- **Style:** строка моно-потока во всю ширину (`100vw`), Geist Mono `1.875rem`, граница 2px (`--flow-line-border`). Завершённые символы прижаты вправо, ожидающие — влево; курсор посередине (режимы HALF / THIRD / QUARTER / DINAMIC).
- **Press-result фон:** `CORRECT` — мягкий зелёный фон, `ERROR` — мягкий красный, `NONE` — прозрачный.

### CursorSymbol
- **Bar:** курсор RECTANGLE / VERTICAL / UNDERSCORE; появляется через `opacity 0.1s`. Символ под курсором использует `mix-blend-mode: difference`, чтобы оставаться читаемым на любой заливке.

### Landing CTA
- **Style:** заливка янтарём цели, тёмный графитовый текст, скругление 6px. Hover — чуть более тёплый янтарь. CTA = «куда двигаться», тот же янтарь, что у цели на клавиатуре.

## 6. Do's and Don'ts

### Do:
- **Do** держать весь хром нейтральным, в оттенках серого (chroma 0); отдавать насыщенность только визуализации движения, пальцам и клавишам.
- **Do** сохранять постоянный оттенок за каждым пальцем во всех четырёх темах — цвет здесь несёт смысл.
- **Do** набирать печатаемый поток моноширинным Geist Mono, а UI — пропорциональным Geist.
- **Do** держать поверхности плоскими; глубину выражать границей 1px или кольцом-обводкой.
- **Do** проверять контраст текста и ключевых состояний ≥4.5:1 во всех четырёх темах (см. PRODUCT.md → Accessibility).
- **Do** обеспечивать различимость пальцев/состояний не только оттенком (позиция, форма, паттерн, надпись) — ради дальтонизма.
- **Do** давать каждому интерактивному элементу состояния default / hover / focus-visible / active; focus всегда видим.

### Don't:
- **Don't** скатываться в громкую геймификацию (Duolingo-стиль): маскоты, серии, конфетти, поздравления на каждом шагу. Награда здесь тихая.
- **Don't** делать стерильный корпоративный SaaS: navy + серый, безликие одинаковые карточки, «ещё одна корпоративная панель».
- **Don't** повторять перегруженных конкурентов (typing.com / ratatype): busy-панели статистики, реклама, визуальный шум по периметру тренировки.
- **Don't** красить хром (кнопки, поля, навигацию, заголовки) — насыщенность принадлежит движению.
- **Don't** использовать `clamp()` / fluid-типографику в продуктовом UI; шкала фиксированная.
- **Don't** ставить размытые drop-shadow на клавиши и плоские панели; `box-shadow` на клавишах — только кольцо-обводка роли.
- **Don't** применять боковые цветные полосы (`border-left`/`right` > 1px как акцент), gradient-text (`background-clip: text` по градиенту) и декоративные эффекты матового стекла — запрещены.
