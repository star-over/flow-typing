# Задачи для реализации

- Реализовать несколько варинтов FlowLine
  1. Это обычный FlowLine, targetSymbol находится в центре и стоит на месте, а строка движется справо налево. Строка движется монотонным потоком, без переносов новых строк и т.п.
  2. FlowLine**s** - по виду как обычный textInput но курсор двигается как при обычном вводе текста

- Реализовать несколько вариантов реагирования на ошибку
  1. Ошибочный символ пропускается, продолжаем печатать дальше. Но можно вернуться назад, например с помощью backspace и исправить символ. Более приближен к поведению при реальной печати, с возможностью исправления.
  2. При Ошибке движение тормозится до тех пор пока символ не будет исправлен, более хардкорный режим

- Добавить в src/data/keyboard-layout-ansi.ts свойство отражающее тип
  - Символ
    - 0-9, a-z
  - Модификатор
    - shift, alt, ctrl, command
  - Управление
    - Enter, Backspace, Delete, ArrowUP (Left, Right, Down), Tab, Escape
  Данный модификатор должен срабатывать в фильтрации клавиш при их мапинге
  А так же при проверке текстов на соответствие раскладки. Например название клавиши `tab` может быт воспринято программой проверки как часть символ в тексте, и проверка в тексте пройдет не корректно.
  Так же, лучше завязать цветовое кодирование клавиш не на эфемерный ColorGroup а на тип системный тип клавиши.
